SYS-2. 스토리지 포맷 (Segment v2)

계층 구조

graph TD T["Tablet"] T --> RS["RowSet (cumulative or base)"] RS --> SEG["Segment 파일 (.dat)"] SEG --> CB["Column Block"] CB --> PG["Page (encoded + compressed)"]

Segment 파일

한 Segment는 컬럼 단위 데이터를 담은 바이너리 파일. 다음 메타가 함께 저장된다.

  • Footer — 스키마, 컬럼 메타, 인덱스 위치.
  • Zone Map — 페이지 단위 min/max/null 통계.
  • Bitmap / Bloom Filter / N-Gram 인덱스 (사용자 지정 시).
  • Short Key Index — 정렬키 prefix.

Encoding

인코딩적용
BIT_SHUFFLE정수/실수의 칼럼 단위 비트 셔플 — 압축률 향상
RLE반복값 (예: 같은 status 연속)
DICTIONARY저카디널리티 STRING — id로 변환
PLAIN일반 저장

Compression

Encoding 후 페이지 단위 압축. 코덱은 LZ4(기본) / ZSTD / SNAPPY / ZLIB.

코덱속도압축률권장
LZ4가장 빠름중간핫 테이블 기본
ZSTD빠름높음대용량 콜드 / 로그
SNAPPY빠름중간—
ZLIB느림높음레거시

Low-Cardinality 최적화

저카디널리티 STRING은 자동으로 사전(Dictionary) 인코딩 + 정수 비교로 동작. cbo_enable_low_cardinality_optimize=true이면 GROUP BY/ORDER BY가 정수 처리로 매우 빨라짐.

Page Cache

BE는 디스크에서 읽은 페이지를 메모리에 캐시. 같은 페이지가 다음 쿼리에서도 재사용. storage_page_cache_limit로 크기 제어.

필터 가지치기 흐름

  1. Partition 가지치기 — 파티션 키 WHERE
  2. Tablet 가지치기 — 분배 키 WHERE
  3. Segment Footer의 컬럼 통계 — 전체 segment 제외 가능
  4. Page-level Zone Map / Bloom — 페이지 제외
  5. Bitmap/Inverted/N-Gram — row 단위 ID set 좁힘
  6. 실제 페이지 디코딩 + 술어 평가
클러스터 적용 노트
  • 저카디널리티 STRING(상태/카테고리)은 가능한 한 STRING 그대로 둬도 SR이 dictionary 처리 — INT로 굳이 변환 안 해도 됨.
  • 정렬키 첫 컬럼은 가장 자주 가지치기되는 컬럼으로. 36바이트 제한.
  • storage_page_cache_limit은 약 20~30%가 일반적. 256GB BE에서 50~75GB.

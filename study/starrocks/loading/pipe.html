Pipe — 자동 감지 적재 (S3 / HDFS)

📌 한 줄 요약

한 번 등록하면 SR이 S3/HDFS의 새 파일을 자동 감지·적재한다. INSERT INTO ... SELECT FROM FILES()의 자동화 버전. 100GB ~ TB 규모 배치를 잘게 끊어 처리.

언제 쓰나

  • 외부 시스템이 S3/HDFS에 정기적으로 파일을 떨어뜨리는 환경.
  • "새 파일이 도착하면 SR에 적재"를 자동화하고 싶을 때.
  • 대용량 배치를 한 번에 INSERT 하기 부담스러울 때 — Pipe가 자동으로 작은 단위로 분할.

CREATE PIPE

CREATE PIPE [IF NOT EXISTS] pipe_name
PROPERTIES (
  "AUTO_INGEST"   = "TRUE",       -- 자동 폴링·적재
  "POLL_INTERVAL" = "30",         -- 초 단위
  "BATCH_SIZE"    = "1GB",        -- 한 번 처리할 크기
  "BATCH_FILES"   = "100"         -- 한 번 처리할 파일 수
)
AS
INSERT INTO <target_table>
SELECT * FROM FILES (
  "path"   = "s3://bucket/events/*.parquet",
  "format" = "parquet",
  "aws.s3.endpoint"   = "...",
  "aws.s3.access_key" = "...",
  "aws.s3.secret_key" = "...",
  "aws.s3.enable_path_style_access" = "true"
);

이미 처리한 파일은 메타에 기록 → 같은 파일 두 번 처리 안 됨 (멱등).

관리 명령

SHOW PIPES;
SHOW PIPES FROM mart;
DESC PIPE pipe_name;

-- 일시 정지 / 재개 / 삭제
ALTER PIPE pipe_name SUSPEND;
ALTER PIPE pipe_name RESUME;
DROP PIPE pipe_name;

-- 옵션 변경
ALTER PIPE pipe_name SET ("BATCH_SIZE" = "2GB");

-- 처리한 파일 / 진행 상태
SELECT * FROM information_schema.pipes WHERE name = 'pipe_name';
SELECT * FROM information_schema.pipe_files WHERE pipe_name = 'pipe_name'
ORDER BY load_state, last_modified DESC LIMIT 50;

실전 예 — 일자별 이벤트 파일 자동 적재

-- 1) 타깃 테이블
CREATE TABLE mart.events (
  event_time DATETIME NOT NULL,
  user_id    BIGINT,
  event_type VARCHAR(32),
  payload    JSON,
  dt         DATE AS (CAST(event_time AS DATE))
)
DUPLICATE KEY(event_time, user_id)
PARTITION BY dt
DISTRIBUTED BY RANDOM
PROPERTIES ("replication_num" = "3", "compression" = "ZSTD");

-- 2) Pipe 생성
CREATE PIPE mart.events_pipe
PROPERTIES (
  "AUTO_INGEST" = "TRUE",
  "POLL_INTERVAL" = "60",
  "BATCH_SIZE" = "1GB"
)
AS
INSERT INTO mart.events (event_time, user_id, event_type, payload)
SELECT event_time, user_id, event_type, payload
FROM FILES (
  "path"   = "s3://datalake/events/dt=*/*.parquet",
  "format" = "parquet",
  "aws.s3.endpoint"   = "https://minio.internal:9000",
  "aws.s3.access_key" = "...",
  "aws.s3.secret_key" = "...",
  "aws.s3.enable_path_style_access" = "true"
);

-- 3) 모니터링
SHOW PIPES;
SELECT pipe_name, file_name, load_state, error_msg
FROM information_schema.pipe_files
WHERE pipe_name = 'events_pipe' AND load_state = 'ERROR';

주의

  • 처리한 파일 메타가 SR에 보관됨 — 외부에서 같은 경로의 파일을 덮어쓰면 재처리 안 됨.
  • 실패 파일은 load_state = 'ERROR'로 남음. 원인 점검 후 수동 재시도.
  • FILES()와 같은 권한·인증 — IAM/Access Key 부여.
  • v3.2+ 기능. 사용 SR 버전 확인.
클러스터 적용 노트
  • 외부 시스템이 일별/시간별 parquet 파일을 떨어뜨리는 환경에 가장 잘 맞는 적재 방식. 운영자 개입 거의 없음.
  • 처리 큐가 너무 길어지면 BATCH_SIZE/POLL_INTERVAL 조정.
  • 실패 파일 재처리는 일반적으로 ALTER PIPE ... RETRY(있는 버전) 또는 SR 측 메타 정리 후 재등록.

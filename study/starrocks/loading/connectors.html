외부 도구에서 SR로 직접 적재하는 4가지 커넥터. Stream Load HTTP 호출을 표준화해 SDK처럼 사용.

Kafka Connect Sink Connector

Kafka Connect 클러스터에서 SR Sink Connector를 등록하면 Kafka 토픽 데이터가 자동으로 SR에 적재된다. Routine Load와 비교해 Kafka Connect 표준 도구 생태계와 통합이 장점.

Flink Connector

Flink SQL 또는 DataStream API에서 sink로 SR을 사용. Routine Load보다 변환·조인이 강력. 자세한 코드 예시.

CREATE TABLE sr_orders (
  order_id BIGINT, user_id BIGINT, status STRING,
  amount DECIMAL(18,2), updated_at TIMESTAMP(3)
) WITH (
  'connector' = 'starrocks',
  'jdbc-url'  = 'jdbc:mysql://fe1:9030',
  'load-url'  = 'fe1:8030;fe2:8030',
  'database-name' = 'mart', 'table-name' = 'orders',
  'username' = 'etl', 'password' = '...',
  'sink.properties.format' = 'json',
  'sink.semantic' = 'exactly-once'
);

INSERT INTO sr_orders SELECT * FROM kafka_source;

Flink CDC + SR 조합은 OLTP → SR 실시간 동기화의 표준 패턴.

Spark Connector

Spark에서 SR을 데이터 소스로 읽거나 쓰기. 자세한 코드 예시.

// 읽기
val df = spark.read.format("starrocks")
  .option("starrocks.fenodes","fe1:8030,fe2:8030")
  .option("starrocks.table.identifier","mart.orders")
  .option("starrocks.user","etl")
  .option("starrocks.password","...")
  .load()

// 쓰기 (Stream Load 기반)
df.write.format("starrocks")
  .option("starrocks.fenodes","fe1:8030,fe2:8030")
  .option("starrocks.table.identifier","mart.orders")
  ...
  .mode("append").save()

SMT (StarRocks Migration Tool)

MySQL/PostgreSQL/Oracle 등을 Flink 경유로 SR에 동기화하는 도구. 스키마 자동 매핑 + Flink CDC 잡 자동 생성.

# 일반 흐름
1. SMT 설정 파일 작성 (소스 RDB 정보, 대상 SR 정보, 테이블 매핑)
2. SMT 실행 → 다음을 자동 생성:
   - SR DDL (CREATE TABLE)
   - Flink job SQL (CDC 소스 + SR sink)
3. 생성된 SQL을 Flink 클러스터에 제출

# 결과
- 초기 스냅샷 + 실시간 변경 동기화
- PK 테이블 자동 UPSERT/DELETE

SMT는 이전(migration) + 동기화를 한 번에 처리하는 헬퍼. CDC 셋업을 직접 짜기 부담스러울 때 유용.

선택 가이드

상황추천
Kafka 토픽 → SR (단순)Routine Load
Kafka 토픽 → SR (Connect 생태계)Kafka Connect Sink
Flink 변환 후 SR로Flink Connector
Spark 배치 변환 후 SR로Spark Connector 또는 Spark Load
MySQL/PG → SR 실시간 동기화 시작SMT (Flink CDC 기반)
이미 있는 ETL/오케스트레이션과 통합해당 도구의 SR 어댑터(dbt-starrocks 등)
클러스터 적용 노트
  • SR 단독 ETL이 목표라면 Routine Load + Pipe로 대부분 커버 가능. Connector는 외부 변환 도구가 이미 있을 때만 도입.
  • Flink CDC + SR PK 테이블이 OLTP 동기화의 표준 — SMT가 이걸 자동 생성해 빠르게 시작 가능.

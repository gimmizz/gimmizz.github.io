L-1. Stream Load

📌 한 줄 요약

HTTP PUT으로 한 파일을 한 트랜잭션으로 적재. 가장 단순하고 자동화하기 쉬워 ETL 도구가 흔히 사용한다.

기본 호출

curl --location-trusted -u user:passwd \
     -H "label:orders_20260507_001" \
     -H "format:CSV" \
     -H "column_separator:," \
     -H "columns:order_id,user_id,status,amount,updated_at" \
     -T orders.csv \
     http://fe1:8030/api/mart/orders/_stream_load

JSON 적재

curl --location-trusted -u user:passwd \
     -H "label:events_20260507_001" \
     -H "format:JSON" \
     -H "strip_outer_array:true" \
     -H "jsonpaths:[\"$.event_time\",\"$.user_id\",\"$.event_type\"]" \
     -H "columns:event_time,user_id,event_type" \
     -T events.json \
     http://fe1:8030/api/mart/events/_stream_load

UPSERT / DELETE (Primary Key 테이블)

# UPSERT (기본)
-H "columns:order_id,user_id,status,amount,updated_at"

# DELETE (해당 row 삭제)
-H "merge_type:DELETE"
-H "columns:order_id"

# MERGE (열별 PARTIAL UPDATE도 가능)
-H "partial_update:true"
-H "columns:order_id,status"

자주 쓰는 옵션

헤더설명
label중복 방지 식별자. 필수 권장
formatCSV / JSON / Parquet / ORC
column_separatorCSV 구분자 (예: ,)
columns적재할 컬럼 순서 / 표현식
where적재 시 필터링
max_filter_ratio허용 실패 비율 (0~1)
timeout초 단위

결과 확인

SHOW STREAM LOAD WHERE LABEL='orders_20260507_001'\G

-- 또는 응답 JSON의 ErrorURL을 따라가면 실패 행이 적힘
클러스터 적용 노트
  • FE 어디로 보내도 OK이지만, 사내 LB 뒤에 두면 단일 진입점이 깔끔.
  • label은 {table}_{yyyymmddHHMMSS}_{idx}가 추적 용이.
  • 실패 행이 자주 나오는 ETL은 max_filter_ratio를 살짝 열어 두되, 모니터링에서 추적.

L-2. Routine Load (Kafka)

📌 한 줄 요약

StarRocks가 직접 Kafka 토픽을 구독하고 micro-batch 단위로 자동 적재. CDC/이벤트 스트림 표준 적재 방식.

잡 생성 (CSV)

CREATE ROUTINE LOAD mart.events_kafka ON events
COLUMNS (event_time, user_id, event_type, payload)
PROPERTIES (
  "format" = "CSV",
  "column_separator" = ",",
  "max_batch_interval" = "10",      -- 초
  "max_batch_rows"     = "200000",
  "max_batch_size"     = "104857600", -- 100MB
  "max_error_number"   = "10000"
)
FROM KAFKA (
  "kafka_broker_list"  = "kafka1:9092,kafka2:9092",
  "kafka_topic"        = "events_v1",
  "kafka_partitions"   = "0,1,2,3",
  "kafka_offsets"      = "OFFSET_BEGINNING,OFFSET_BEGINNING,OFFSET_BEGINNING,OFFSET_BEGINNING"
);

JSON 적재 (CDC 형식 일반적)

CREATE ROUTINE LOAD mart.orders_cdc ON orders
COLUMNS (order_id, user_id, status, amount, updated_at)
PROPERTIES (
  "format" = "JSON",
  "jsonpaths" = "[\"$.id\",\"$.user_id\",\"$.status\",\"$.amount\",\"$.updated_at\"]",
  "max_batch_interval" = "5",
  "max_batch_rows"     = "100000"
)
FROM KAFKA (
  "kafka_broker_list" = "kafka:9092",
  "kafka_topic"       = "cdc.orders"
);

운영 SQL

SHOW ROUTINE LOAD\G
SHOW ROUTINE LOAD FOR mart.events_kafka\G

-- 일시 정지 / 재개 / 종료
PAUSE  ROUTINE LOAD FOR mart.events_kafka;
RESUME ROUTINE LOAD FOR mart.events_kafka;
STOP   ROUTINE LOAD FOR mart.events_kafka;

-- 옵션 변경 (PAUSE 상태에서)
ALTER ROUTINE LOAD mart.events_kafka
PROPERTIES ("max_batch_rows" = "300000");

Kafka SASL/SSL 인증

FROM KAFKA (
  "kafka_broker_list" = "kafka:9093",
  "kafka_topic"       = "topic_name",
  "property.security.protocol" = "SASL_SSL",
  "property.sasl.mechanism"    = "SCRAM-SHA-256",
  "property.sasl.username"     = "starrocks",
  "property.sasl.password"     = "..."
)
클러스터 적용 노트
  • 약 17 BE 환경에서 한 잡의 파티션 수는 BE 수와 비슷하거나 그 배수일 때 안정적.
  • 잡이 PAUSED로 멈춰 있는지 알람 필수 — 무한히 처리 안 되는 일이 잦음.
  • Primary Key 테이블에 적재하면 자동 UPSERT/DELETE — CDC 적재 표준 패턴.

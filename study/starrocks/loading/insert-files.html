L-4. INSERT INTO ... SELECT · FILES() 함수

📌 한 줄 요약

SQL 한 줄로 적재한다. 외부 카탈로그가 있으면 SELECT, 파일만 있으면 FILES() 테이블 함수.

외부 카탈로그 → 내부 마트

INSERT INTO default_catalog.mart.events
SELECT event_time, user_id, event_type, payload
FROM hive_prod.dwh.events
WHERE dt = '2026-05-07';

FILES() 테이블 함수 (v3.x)

-- HDFS/S3의 parquet 파일을 즉시 SELECT
SELECT count(*)
FROM FILES(
  "path"   = "s3://bucket/path/*.parquet",
  "format" = "parquet",
  "aws.s3.endpoint"   = "https://minio.internal:9000",
  "aws.s3.access_key" = "...",
  "aws.s3.secret_key" = "...",
  "aws.s3.enable_path_style_access" = "true"
);

-- 그대로 적재
INSERT INTO mart.events
SELECT *
FROM FILES(
  "path"   = "s3://bucket/path/*.parquet",
  "format" = "parquet",
  "aws.s3.endpoint" = "..."
);

-- CSV / JSON 도 가능
SELECT *
FROM FILES("path"="hdfs://nn/tmp/data.json", "format"="json");

CTAS — 즉석 마트 생성

CREATE TABLE mart.snap_users
PROPERTIES ("replication_num" = "3")
AS
SELECT user_id, count(*) AS cnt
FROM hive_prod.dwh.events
WHERE dt = '2026-05-07'
GROUP BY user_id;
클러스터 적용 노트
  • FILES()는 단발 처리에 가장 편함: 분석가가 파일만 받아서 즉시 검증할 때 진가.
  • 정기 ETL은 INSERT INTO + SELECT: 외부 카탈로그를 한 번 만들어 두면 그 위에 SQL만 짜면 됨. Async MV로 이 로직을 자동화하면 ETL 코드가 거의 사라짐.

L-3. Broker Load (HDFS / S3 대용량 일괄)

📌 한 줄 요약

HDFS 또는 S3에 있는 대용량 파일을 한 번에 적재하는 비동기 잡. 큰 백필이나 일자별 풀 적재에 적합.

예시 — HDFS의 parquet 적재

LOAD LABEL mart.events_20260507 (
  DATA INFILE("hdfs://nn/warehouse/staging/events/dt=2026-05-07/*")
  INTO TABLE events
  FORMAT AS "parquet"
  (event_time, user_id, event_type, payload)
  WHERE event_type IN ('view','click')
)
WITH BROKER hdfs_broker (
  "username" = "sr",
  "password" = ""
)
PROPERTIES (
  "timeout" = "7200",
  "max_filter_ratio" = "0.01"
);

S3

LOAD LABEL mart.s3_load_001 (
  DATA INFILE("s3a://bucket/path/*.parquet")
  INTO TABLE events
  FORMAT AS "parquet"
)
WITH BROKER (
  "fs.s3a.access.key" = "...",
  "fs.s3a.secret.key" = "...",
  "fs.s3a.endpoint"   = "https://minio.internal:9000"
);

상태 / 취소

SHOW LOAD WHERE LABEL = 'mart.events_20260507'\G

-- 진행 중 잡 취소
CANCEL LOAD FROM mart WHERE LABEL = 'mart.events_20260507';

주의사항

  • Broker가 별도 프로세스(필요시)인 경우 deploy/health 모니터링 필요. 최근 버전은 BE 내장 broker가 일반적.
  • HDFS HA·Kerberos는 Hive Catalog와 같은 사전 준비 필요(참고).
클러스터 적용 노트
  • 이미 Hive Catalog를 붙였다면 INSERT INTO mart.events SELECT * FROM hive_prod.dwh.events WHERE dt='2026-05-07'가 더 간단할 때가 많음. Broker Load는 외부 카탈로그를 안 만든 상태에서의 단발 백필에 유용.
  • 대용량 잡은 새벽에. ETL 전용 Resource Group으로 격리.

2. 배치 사이즈 가이드

  • 너무 작은 batch (1초·수백 행)는 Compaction 적체를 유발.
  • 너무 큰 batch는 한 번 실패 시 재시도 비용이 큼.
  • 일반 권장: 한 번에 수만 ~ 수십만 행, 5~30초 주기.
-- Routine Load 권장 시작값
"max_batch_interval" = "10",         -- 10초
"max_batch_rows"     = "200000",     -- 20만 행
"max_batch_size"     = "104857600"   -- 100MB

3. label 표준

{table}_{yyyymmddHHMMSS}_{run_id}
# 예: orders_20260507093000_a1b2c3

ETL 도구의 task instance ID(Airflow run_id, Dagster run_key)를 끼워 넣으면 자동 멱등.

4. Resource Group 분리

CREATE RESOURCE GROUP rg_etl
WITH ('cpu_core_limit'='24','mem_limit'='50%','concurrency_limit'='8','type'='normal');

ALTER RESOURCE GROUP rg_etl ADD (USER='etl_user');

BI 사용자와 ETL을 같은 RG에 두지 말 것 — 적재 폭주 시 BI 응답이 무너진다.

5. Stage → Mart 분리

  • 입력을 Mart에 직접 쓰지 말고 Stage 테이블에 raw 보존.
  • Stage는 Duplicate + RANDOM + ZSTD + 30~90일 보존(dynamic_partition).
  • Mart는 Primary Key + 분배 키 정렬 + 180일+ 보존.
  • 자세한 설계는 ETL 아키텍처.

6. 스키마 설계

  • Generated Columns로 JSON 핫 키를 컬럼화 (참고).
  • 날짜는 Expression Partition (date_trunc('day', ts)) 권장 — 자동 파티션 생성.
  • 분배 키는 JOIN/필터에 자주 쓰는 컬럼 (보통 PK).
  • 고카디널리티 ID 컬럼은 bloom_filter_columns에 추가.
  • 저카디널리티(상태/카테고리)는 BITMAP 인덱스.

8. 멱등 백필 패턴

# 같은 날 다시 돌려도 결과가 동일하게 (예: dt='2026-05-07' 백필)
INSERT OVERWRITE mart.orders_d PARTITION (dt='2026-05-07')
SELECT date_trunc('day', updated_at) AS dt, user_id, status, count(*), sum(amount)
FROM mart.orders WHERE updated_at::DATE = '2026-05-07'
GROUP BY 1, 2, 3;

9. 워터마크 / 늦은 데이터

  • 실시간 적재에서 5~10분 늦게 도착하는 이벤트가 있다면 겹치는 윈도로 백필: 
    # 15분 단위 잡: 직전 30분치를 INSERT OVERWRITE
INSERT OVERWRITE ...
WHERE event_time >= NOW() - INTERVAL 30 MINUTE;
  • PK 테이블이면 늦은 도착이 자동 UPSERT — 별도 처리 불필요.

10. Capacity 산정

  • BE 한 대당 적재 처리량은 보통 50~200 MB/s(데이터/포맷 의존).
  • 약 17 BE × 100 MB/s = 1.7 GB/s 정도가 안전 상한.
  • 이 이상이 필요하면 BE 추가 또는 Pipe로 batch 분할.

11. 흔한 안티패턴

  • ❌ 1초 단위 1KB Stream Load — Compaction 적체.
  • ❌ 한 큰 잡으로 1년치 백필 — timeout/failure 비용 큼. 일자 단위로 분할.
  • ❌ ON DUPLICATE KEY UPDATE 시도 — SR은 PK 테이블의 자동 UPSERT로 대체.
  • ❌ ETL 사용자에 ALL 권한 — DELETE/DROP 사고 위험. INSERT 한정 권한.
  • ❌ MV 의존하는 테이블에 큰 INSERT 후 MV 무관심 — stale로 BI 응답 오류.
  • ❌ JSON payload 핫 키를 매번 함수로 추출 — Generated Column으로 빼기.

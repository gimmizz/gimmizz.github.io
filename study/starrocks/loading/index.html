트랜잭션 모델

  • 적재 작업은 label 기반 — 같은 label은 한 번만 성공한다(at-most-once).
  • Stream Load 한 호출 = 하나의 트랜잭션. 부분 성공/실패 없음.
  • Routine Load는 내부에서 batch 단위로 트랜잭션을 끊어 처리.
  • Primary Key 테이블은 적재 시 자동 UPSERT/DELETE 처리.

적재 상태 확인

SHOW LOAD WHERE STATE != 'FINISHED';
SHOW LOAD WHERE LABEL = 'my_label_2026_05_07';
SHOW ROUTINE LOAD;
SHOW STREAM LOAD WHERE STATE = 'FAILED';
클러스터 적용 노트
  • 적재가 BE 메모리/디스크에 부담을 주므로 Resource Group으로 ETL을 분리 (BI와 격리).
  • 대용량 백필은 한 번에 하지 말고 파티션 단위로 나누어 진행 — 실패 재시도가 쉬워짐.
  • label은 {table}_{date}_{seq} 형식 추천 — 충돌 방지 + 추적 용이.

3-2. 인덱스 종류

📌 한 줄 요약

StarRocks는 자동 인덱스(Prefix · Zone Map · Bloom Filter for short keys) + 사용자 정의 인덱스(Bitmap, Bloom Filter, N-Gram Bloom)를 함께 제공한다. 대부분의 경우 자동 인덱스로 충분하고, 특정 쿼리 패턴에 맞춰 추가한다.

자동 인덱스 (별도 설정 불필요)

Prefix Index

정렬키의 앞쪽 컬럼 조합으로 자동 생성. 정렬키 첫 컬럼 필터에 가장 강함. 최대 36바이트.

Zone Map

각 컬럼의 데이터 블록(보통 64K rows)에 대해 min/max를 보관. WHERE 조건이 들어오면 필요 없는 블록을 통째로 스킵.

Bitmap Index

저카디널리티 컬럼(성별, 상태, 카테고리 등)에 효과 큼. 다중 조건 AND/OR가 비트 연산으로 끝나 매우 빠름.

-- 생성
ALTER TABLE orders ADD INDEX idx_status (status) USING BITMAP;

-- 조회
SHOW INDEX FROM orders;

주의: 카디널리티가 높은 컬럼(unique 값이 많은)에 만들면 인덱스 자체가 무거워져 오히려 손해.

Bloom Filter Index

WHERE 등호 조건의 빠른 negative lookup용. "이 값이 이 블록에 없음"을 빠르게 판정.

ALTER TABLE orders SET ("bloom_filter_columns" = "user_id, order_id");

중간~고카디널리티의 등호 필터가 자주 들어오는 컬럼에 적합. 범위(BETWEEN, >) 조건엔 무용.

N-Gram Bloom Filter (LIKE 가속, v3.x)

STRING 컬럼의 LIKE '%foo%' 부분 일치를 가속.

ALTER TABLE logs ADD INDEX idx_msg (message)
USING NGRAMBF (
  ("gram_num" = "3", "bloom_filter_fpp" = "0.05")
);

Inverted Index (v3.x, 일부)

특정 STRING 토큰 검색 가속용. 키워드 검색에 적합.

ALTER TABLE logs ADD INDEX idx_msg_inv (message) USING INVERTED;

클러스터 적용 노트
  • 일단 만들지 마세요: 정렬키 + Zone Map만으로 대부분 빠릅니다. EXPLAIN을 보고 스캔 row가 너무 많을 때 추가.
  • Bitmap은 신중히: 저카디널리티(≤ 수천)일 때만. 사용자/세션 ID 같은 고카디널리티엔 Bloom Filter.
  • 인덱스 추가 후 재구축: 기존 데이터에 즉시 적용되지 않을 수 있음. ALTER TABLE ... BUILD INDEX 또는 컴팩션 통해 적용.

진단 절차 (어떤 에러든)

  1. 전문(full) 에러 메시지 + queryId/loadLabel 수집.
  2. SHOW LOAD WHERE LABEL='...' 또는 SHOW PROFILE FROM '...' 으로 단계별 분석.
  3. FE 로그 (fe.log) 또는 BE 로그 (be.WARNING)에서 같은 시각 메시지 확인.
  4. 알려진 케이스라면 위 표 참고. 새로운 케이스라면 GitHub Issues 검색.

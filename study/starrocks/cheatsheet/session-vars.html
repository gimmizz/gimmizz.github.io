Cheat Sheet — 핵심 세션 변수 30

자주 손대는 세션 변수만 추렸습니다. 글로벌 적용은 SET GLOBAL ..., 세션 한정은 SET ....

실행 엔진 / 병렬도

변수기본의미
enable_pipeline_enginetrue파이프라인 엔진. 끄지 말 것.
pipeline_dop0 (auto)BE 코어 활용 도. 0=자동.
parallel_fragment_exec_instance_num1BE당 fragment 인스턴스 수. 무거운 쿼리에 4~8.
query_timeout300 (s)쿼리 타임아웃.
query_mem_limitBE 메모리 비율한 쿼리 메모리 한도 (bytes).
load_mem_limit0 (auto)적재 잡 메모리 한도.

옵티마이저 / 통계

변수기본의미
cbo_enable_low_cardinality_optimizetrue저카디널리티 dictionary 최적화.
broadcast_row_limit15M이 행수 이하면 Broadcast Join 우선.
enable_global_runtime_filtertrue글로벌 RuntimeFilter.
enable_local_shuffle_aggtrueBE 내 로컬 셔플 집계.
cbo_max_reorder_node_use_exhaustive4완전 탐색 조인 순서 한도.
statistic_collect_parallel1ANALYZE 병렬도.

Materialized View

변수기본의미
enable_materialized_view_rewritetrueMV 자동 재작성.
materialized_view_rewrite_modedefault"force_or_default" 등 모드.
nested_mv_rewrite_max_level3중첩 MV 깊이.
force_external_table_query_rewritefalse외부 카탈로그도 강제 rewrite.

External Catalog

변수기본의미
enable_query_cachefalseFE 쿼리 결과 캐시.
enable_data_cachetrueBE Data Cache 사용.
connector_io_tasks_per_scan_operator16외부 카탈로그 스캔 IO 작업 수.
plan_modeauto외부 카탈로그 플랜 모드.

프로파일·디버깅

변수기본의미
enable_profilefalse쿼리 프로파일 기록.
big_query_profile_threshold0이 시간 초과 쿼리 자동 프로파일.
runtime_profile_report_interval10프로파일 리포트 주기 (s).

Spill (디스크 보조)

변수기본의미
enable_spillfalse큰 연산자 디스크 spill 활성.
spill_modeautoauto / force.
spill_storage_limit—spill 디스크 사용 한도.

기타

변수기본의미
time_zonesystem세션 타임존. 'Asia/Seoul' 등.
sql_mode'ONLY_FULL_GROUP_BY,STRICT_TRANS_TABLES' 등.
character_set_resultsutf8mb4결과 인코딩.
query_cache_size0FE 쿼리 캐시 크기.
resource_group—현재 세션 Resource Group.

자주 쓰는 조합

-- 분석가 워크벤치 (큰 쿼리도 안전하게)
SET pipeline_dop = 0;
SET parallel_fragment_exec_instance_num = 8;
SET query_mem_limit = 32 * 1024 * 1024 * 1024;
SET enable_spill = true;
SET enable_profile = true;

-- BI 대시보드 (낮은 지연 우선)
SET query_timeout = 30;
SET query_mem_limit = 8 * 1024 * 1024 * 1024;
SET enable_materialized_view_rewrite = true;

조회·검색

SHOW VARIABLES;
SHOW VARIABLES LIKE '%mv%';

클러스터 적용 노트
  • 운영용 글로벌 변수는 SET GLOBAL로 박아두고 git에 SQL로 저장 (변경 이력 보존).
  • 큰 ad-hoc 쿼리 사용자에게는 Resource Group + 세션 프로필을 미리 설정해서 자동 적용.

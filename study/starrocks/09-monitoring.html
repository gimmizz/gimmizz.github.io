Grafana 대시보드

StarRocks 공식 GitHub의 contrib/grafana-dashboards 또는 커뮤니티 대시보드(grafana.com/dashboards)에서 시작.

대시보드 그룹은 다음으로 분리하면 한눈에 보기 좋음:

  • Overview: 클러스터 alive, QPS, p99 latency.
  • FE: JVM, edit log, connection.
  • BE: 메모리/디스크/compaction/tablet.
  • Loading: Stream/Routine/Broker Load 진행/실패.
  • External Catalog: HMS 호출, Data Cache hit.

Audit Log를 분석에 활용

-- audit_loader 플러그인 적재 후
SELECT user, count(*) AS cnt, avg(query_time)/1000 AS avg_sec
FROM starrocks_audit_db__.starrocks_audit_tbl__
WHERE event_time >= NOW() - INTERVAL 1 HOUR
GROUP BY user ORDER BY cnt DESC LIMIT 20;
클러스터 적용 노트
  • 알람을 너무 민감하게 잡지 말 것: 약 20대 환경에서 BE 1대 재기동 = 잠시 unhealthy tablet 증가. 5분 지속 조건을 두는 게 좋음.
  • 디스크별 모니터링 필수: SSD 8개 중 1개만 가득 차도 BE가 self-protect로 멈출 수 있음.
  • Audit log를 자체 분석: 사용자별 사용량 + 쿼리 타입 분포가 보이면 Resource Group 설계가 자연스럽게 정해짐.

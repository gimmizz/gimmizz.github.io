P-4. Async MV 실습 — Query Rewrite 확인까지

📌 목표

Hive Catalog 위에 Async MV를 만들고, 사용자 SQL이 자동으로 MV로 재작성되는지 EXPLAIN으로 확인한다.

전제

P-3 실습이 끝나 hive_local.dwh.events 테이블이 있는 상태.

Async MV 생성 
USE default_catalog.shop;

CREATE MATERIALIZED VIEW mv_event_user_d
PARTITION BY dt
DISTRIBUTED BY HASH(user_id)
REFRESH ASYNC EVERY (INTERVAL 10 MINUTE)
PROPERTIES (
  "replication_num" = "1",
  "enable_query_rewrite" = "true"
)
AS
SELECT
  date_trunc('day', event_time) AS dt,
  user_id,
  event_type,
  count(*) AS cnt
FROM hive_local.dwh.events
GROUP BY 1, 2, 3;
최초 리프레시 강제 
REFRESH MATERIALIZED VIEW mv_event_user_d;

-- 진행 상황
SHOW MATERIALIZED VIEW mv_event_user_d\G
원본 SQL을 실행하면서 EXPLAIN으로 재작성 확인 
SET enable_materialized_view_rewrite = true;

EXPLAIN
SELECT
  date_trunc('day', event_time) AS dt,
  user_id,
  count(*) AS cnt
FROM hive_local.dwh.events
WHERE event_time >= '2026-05-07'
GROUP BY 1, 2;

플랜의 SCAN 부분에 RewriteByMV: mv_event_user_d 비슷한 표시가 있어야 성공.

비교 — MV 재작성 끄기 
SET enable_materialized_view_rewrite = false;
EXPLAIN SELECT ...;

이번에는 hive_local.dwh.events를 직접 스캔하는 플랜이 나오는지 확인.

MV 활용 응답 시간 비교 
SET enable_materialized_view_rewrite = true;
SELECT count(*) FROM hive_local.dwh.events WHERE dt='2026-05-07';

SET enable_materialized_view_rewrite = false;
SELECT count(*) FROM hive_local.dwh.events WHERE dt='2026-05-07';
클러스터 적용 노트

운영 약 20대 환경에서는 BI 대시보드 SQL 패턴 톱-N를 추출 → MV 후보로 일괄 등록 → Query Rewrite로 자동 가속하는 패턴이 효과가 가장 큼. 사용자 코드 변경이 필요 없다는 점이 핵심 가치.

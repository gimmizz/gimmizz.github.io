P-2. 첫 테이블 · Stream Load 실습

📌 목표

Primary Key 테이블을 만들고, CSV를 Stream Load로 적재한 뒤 UPSERT/DELETE까지 확인한다.

테이블 생성 
CREATE DATABASE IF NOT EXISTS shop;
USE shop;

CREATE TABLE orders (
  order_id   BIGINT NOT NULL,
  user_id    BIGINT,
  status     VARCHAR(32),
  amount     DECIMAL(18,2),
  updated_at DATETIME
)
PRIMARY KEY(order_id)
PARTITION BY date_trunc('day', updated_at)
DISTRIBUTED BY HASH(order_id)
PROPERTIES (
  "replication_num" = "1",          -- 단일 노드 데모
  "enable_persistent_index" = "true"
);
샘플 CSV 만들기 
cat > /tmp/orders.csv <<EOF
1,100,paid,12345.00,2026-05-07 10:00:00
2,101,pending,5000.00,2026-05-07 10:01:00
3,100,paid,890.00,2026-05-07 10:02:00
EOF
Stream Load로 적재 
curl --location-trusted -u root: \
  -H "label:orders_init_001" \
  -H "format:CSV" \
  -H "column_separator:," \
  -H "columns:order_id,user_id,status,amount,updated_at" \
  -T /tmp/orders.csv \
  http://127.0.0.1:8030/api/shop/orders/_stream_load

응답 JSON에서 "Status":"Success" 확인.

SELECT 결과 확인 
USE shop;
SELECT * FROM orders ORDER BY order_id;
UPSERT 적용 — 같은 PK로 다시 적재 
cat > /tmp/orders_update.csv <<EOF
1,100,refunded,0.00,2026-05-07 11:00:00
EOF

curl --location-trusted -u root: \
  -H "label:orders_upd_001" \
  -H "format:CSV" \
  -H "column_separator:," \
  -H "columns:order_id,user_id,status,amount,updated_at" \
  -T /tmp/orders_update.csv \
  http://127.0.0.1:8030/api/shop/orders/_stream_load

# 다시 SELECT — order_id=1이 'refunded'로 갱신됐는지 확인
SELECT * FROM orders WHERE order_id = 1;
DELETE 적용 
cat > /tmp/orders_del.csv <<EOF
2
EOF

curl --location-trusted -u root: \
  -H "label:orders_del_001" \
  -H "format:CSV" \
  -H "column_separator:," \
  -H "columns:order_id" \
  -H "merge_type:DELETE" \
  -T /tmp/orders_del.csv \
  http://127.0.0.1:8030/api/shop/orders/_stream_load

SELECT count(*) FROM orders;  -- 2건 남음
적재 이력 확인 
SHOW STREAM LOAD FROM shop\G
클러스터 적용 노트

운영 환경에서 Stream Load는 ETL 도구(Airflow Operator, Kettle, 자체 스크립트)에 통합. label은 {table}_{date}_{seq}로 표준화. 실패 시 ErrorURL을 ETL 로그에 기록.

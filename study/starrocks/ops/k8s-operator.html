K8s Operator · Helm 배포

📌 한 줄 요약

StarRocks Operator + Helm chart로 K8s 위에 클러스터를 선언적으로 운영. 클라우드 또는 K8s 표준화된 사내 환경에서 유리.

Operator 설치

# Helm repo 추가
helm repo add starrocks https://starrocks.github.io/starrocks-kubernetes-operator
helm repo update

# Operator 설치
helm install starrocks-operator starrocks/operator \
  -n starrocks --create-namespace

StarRocksCluster CRD

# sr-cluster.yaml
apiVersion: starrocks.com/v1
kind: StarRocksCluster
metadata:
  name: sr-prod
  namespace: starrocks
spec:
  starRocksFeSpec:
    replicas: 3
    image: starrocks/fe-ubuntu:4.1.0
    requests:
      cpu: 4
      memory: 16Gi
    limits:
      cpu: 8
      memory: 32Gi
    storageVolumes:
      - name: fe-meta
        storageClassName: fast-ssd
        storageSize: 100Gi
        mountPath: /opt/starrocks/fe/meta
  starRocksBeSpec:
    replicas: 5
    image: starrocks/be-ubuntu:4.1.0
    requests:
      cpu: 16
      memory: 64Gi
    limits:
      cpu: 32
      memory: 128Gi
    storageVolumes:
      - name: be-data
        storageClassName: fast-ssd
        storageSize: 1Ti
        mountPath: /opt/starrocks/be/storage
    configMapInfo:
      configMapName: be-conf
      resolveKey: be.conf
kubectl apply -f sr-cluster.yaml

# 상태 확인
kubectl get starrockscluster -n starrocks
kubectl get pods -n starrocks
kubectl logs -f sr-prod-fe-0 -n starrocks

be.conf / fe.conf 주입

kubectl create configmap be-conf -n starrocks \
  --from-file=be.conf=./be.conf

# 변경 후 BE 재시작 트리거
kubectl rollout restart statefulset/sr-prod-be -n starrocks

Service / Ingress

# Operator가 자동으로 만드는 Service:
kubectl get svc -n starrocks
# - sr-prod-fe-service       (MySQL :9030, HTTP :8030)
# - sr-prod-fe-search        (Headless)
# - sr-prod-be-service

# 외부 노출 (LoadBalancer 또는 Ingress)
apiVersion: v1
kind: Service
metadata:
  name: sr-prod-mysql-ext
spec:
  type: LoadBalancer
  selector:
    app.kubernetes.io/name: sr-prod-fe
  ports:
    - port: 9030
      targetPort: 9030

Shared-Data on K8s

K8s 환경에서는 PV 비용 부담 + 탄력성 요구가 크므로 Shared-Data 모드와 잘 어울린다.

spec:
  starRocksFeSpec: { ... }
  starRocksCnSpec:                    # CN spec (BE 대신)
    replicas: 6
    image: starrocks/cn-ubuntu:4.1.0
    storageVolumes:                   # 캐시용 로컬 SSD
      - name: cn-cache
        storageClassName: local-ssd
        storageSize: 500Gi
        mountPath: /opt/starrocks/cn/cache
  storageSpec:
    storageVolume:
      type: S3
      bucket: sr-warehouse
      endpoint: https://s3.example.com
      credential:
        secretName: s3-creds

자동 확장 (HPA)

apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: sr-prod-cn-hpa
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: StatefulSet
    name: sr-prod-cn
  minReplicas: 3
  maxReplicas: 20
  metrics:
    - type: Resource
      resource:
        name: cpu
        target:
          type: Utilization
          averageUtilization: 70

주의

  • BE는 stateful: PVC 보관·노드 친화성 설정 필수.
  • K8s 노드 재기동 = BE 한 대 다운으로 취급 — 한 번에 너무 많이 재기동되지 않게 PodDisruptionBudget 설정.
  • Shared-Nothing K8s에서 Pod IP 변경이 잦으면 FQDN 모드 사용.
  • Ingress보다 LoadBalancer가 MySQL 프로토콜에 자연스럽다 (TCP).
클러스터 적용 노트
  • 온프레 PM 클러스터를 먼저 운영하고, 다음 단계로 클라우드 K8s 환경(또는 자동확장이 필요한 워크로드)에 한해 K8s 도입을 검토.

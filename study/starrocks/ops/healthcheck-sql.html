OPS-2. 헬스체크 SQL — cron으로 돌릴 수 있는 형태

아래 SQL을 5~15분 주기로 cron + Slack/웍스 알림에 묶어두면 사고 감지가 빨라집니다.

노드 alive

-- FE alive 수 (3 이어야 함)
SELECT count(*) AS fe_alive
FROM information_schema.frontends WHERE alive = 'true';

-- BE alive 수 (배포한 BE 수와 같아야 함)
SELECT count(*) AS be_alive
FROM information_schema.backends WHERE alive = 'true';

디스크

-- BE 디스크 사용률 ≥ 80% 인 노드
SELECT BackendId, IP, MaxDiskUsedPct
FROM information_schema.backends
WHERE alive='true' AND MaxDiskUsedPct >= 80;

불건전 Tablet

-- 불건전(미복구/오프라인) tablet 수 — 0 이어야 함
SHOW PROC '/statistic';
-- 또는
SELECT count(*) FROM information_schema.tablet_schedules
WHERE state IN ('PENDING','RUNNING');

컴팩션 적체

-- BE별 max compaction score
SELECT BackendId, IP, TabletNum, MaxCompactionScore
FROM information_schema.backends
ORDER BY MaxCompactionScore DESC LIMIT 10;
-- score > 100 이면 적체 알람

적재 잡

-- 비정상 종료 적재
SELECT label, state, type, error_msg, finish_time
FROM information_schema.loads
WHERE state IN ('CANCELLED','FAILED')
  AND create_time >= NOW() - INTERVAL 1 HOUR;

-- Routine Load PAUSED
SHOW ROUTINE LOAD\G
-- 또는
SELECT name, state, error_log_urls
FROM information_schema.routine_loads
WHERE state = 'PAUSED';

MV 상태

-- 마지막 리프레시가 24시간 이상 지난 MV
SELECT TABLE_NAME, last_refresh_time, last_refresh_state, is_active
FROM information_schema.materialized_views
WHERE is_active = 'true'
  AND last_refresh_time < NOW() - INTERVAL 24 HOUR;

장기 실행 쿼리

-- 30분 이상 도는 쿼리
SELECT id, user, host, db, command, time, state, info
FROM information_schema.processlist
WHERE time > 1800
ORDER BY time DESC;

하나의 헬스체크 스크립트 예

#!/usr/bin/env bash
SR="mysql -h fe1 -P 9030 -u sr_check -p$SR_PWD -N -B -e"

FE_ALIVE=$($SR "SELECT count(*) FROM information_schema.frontends WHERE alive='true'")
BE_ALIVE=$($SR "SELECT count(*) FROM information_schema.backends  WHERE alive='true'")
DISK_BAD=$($SR "SELECT count(*) FROM information_schema.backends WHERE MaxDiskUsedPct >= 90")
COMP_BAD=$($SR "SELECT count(*) FROM information_schema.backends WHERE MaxCompactionScore > 100")
RL_PAUSED=$($SR "SELECT count(*) FROM information_schema.routine_loads WHERE state='PAUSED'")

if [ "$FE_ALIVE" -lt 3 ] || [ "$BE_ALIVE" -lt 17 ] || [ "$DISK_BAD" -gt 0 ] \
   || [ "$COMP_BAD" -gt 0 ] || [ "$RL_PAUSED" -gt 0 ]; then
  echo "ALERT: FE=$FE_ALIVE BE=$BE_ALIVE diskBad=$DISK_BAD compBad=$COMP_BAD rlPaused=$RL_PAUSED" \
    | /opt/notify-works.sh
fi
클러스터 적용 노트
  • 알람은 5분 지속 조건으로 묶어 일시적 노이즈 차단.
  • 전용 read-only 계정(sr_check)을 만들고 information_schema SELECT만 부여.
  • Prometheus 알람과 SQL 헬스체크를 병행하면 사각지대가 거의 없음.

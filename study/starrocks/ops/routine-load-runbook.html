Routine Load 운영 Runbook

잡 상태

State의미대응
NEED_SCHEDULE새로 만들어진 잡, 스케줄 대기자동으로 RUNNING으로 진입
RUNNING정상 적재 중—
PAUSED실패율 한도 도달 또는 수동 정지원인 파악 후 RESUME
STOPPED완전 종료 (재가동 불가)새 잡 생성
CANCELLED운영자가 취소새 잡 생성

상태 확인

SHOW ROUTINE LOAD;
SHOW ROUTINE LOAD FOR mart.orders_cdc\G

-- 최근 처리 통계 / 에러
SHOW ROUTINE LOAD TASK FROM mart\G
SHOW ROUTINE LOAD FOR mart.orders_cdc;
-- ErrorLogUrl 필드 → 실패 행 확인

정지 / 재개 / 종료

PAUSE  ROUTINE LOAD FOR mart.orders_cdc;
RESUME ROUTINE LOAD FOR mart.orders_cdc;
STOP   ROUTINE LOAD FOR mart.orders_cdc;

잡 옵션 변경

일반적으로 PAUSE 상태에서만 가능.

PAUSE ROUTINE LOAD FOR mart.orders_cdc;

ALTER ROUTINE LOAD mart.orders_cdc
PROPERTIES (
  "max_batch_rows"     = "300000",
  "max_batch_interval" = "10"
)
FROM kafka (
  "kafka_partitions" = "0,1,2,3,4,5",
  "kafka_offsets"    = "OFFSET_END,OFFSET_END,OFFSET_END,OFFSET_END,OFFSET_END,OFFSET_END"
);

RESUME ROUTINE LOAD FOR mart.orders_cdc;

오프셋 리셋

처음부터 다시

# 1) 기존 잡 STOP
STOP ROUTINE LOAD FOR mart.orders_cdc;

# 2) 같은 이름으로 재생성 (kafka_offsets = OFFSET_BEGINNING)
CREATE ROUTINE LOAD mart.orders_cdc ON orders ...
FROM KAFKA (
  "kafka_topic" = "cdc.orders",
  "kafka_partitions" = "0,1,2,3",
  "kafka_offsets" = "OFFSET_BEGINNING,OFFSET_BEGINNING,OFFSET_BEGINNING,OFFSET_BEGINNING"
);

특정 시점부터

"kafka_offsets" = "OFFSET_END,OFFSET_END,..."        -- 가장 최근부터
"kafka_offsets" = "12345,12346,12347,12348"          -- 지정 offset

새 Kafka 파티션 추가

토픽이 파티션을 늘렸을 때:

PAUSE ROUTINE LOAD FOR mart.orders_cdc;

ALTER ROUTINE LOAD mart.orders_cdc
FROM kafka (
  "kafka_partitions" = "0,1,2,3,4,5,6,7",          -- 새 파티션 포함
  "kafka_offsets"    = "OFFSET_END,...,OFFSET_BEGINNING,OFFSET_BEGINNING"
);

RESUME ROUTINE LOAD FOR mart.orders_cdc;

자주 만나는 PAUSED 원인

원인증상해법
스키마 불일치too many filtered rowsjsonpaths 점검, 컬럼 타입 확인
Broker 다운Kafka 연결 실패broker 복구 후 RESUME
Auth 만료SASL 인증 실패password/keytab 갱신
토픽 삭제topic not foundSTOP 후 새 잡
BE 메모리 부족load mem limit exceededbatch 크기 ↓, 다른 잡 정리

알람 권장

-- PAUSED 잡 검출 (5분마다 cron)
SELECT name, state, error_log_urls
FROM information_schema.routine_loads
WHERE state = 'PAUSED';
클러스터 적용 노트
  • 잡당 Kafka 파티션 수는 BE 수와 비슷하거나 그 배수일 때 안정적.
  • label 충돌 방지를 위해 잡 이름은 {db}.{table}_kafka_{topic} 형식으로 표준화.
  • PAUSED 알람을 안 거는 건 사고. 거의 모든 운영 사고가 여기서 시작.

6-8. Elasticsearch Catalog

📌 한 줄 요약

v3.1부터 ES 인덱스를 SQL로 SELECT할 수 있다. ES의 검색·필터를 푸시다운하고, 다른 카탈로그와 JOIN도 가능.

PROPERTIES

CREATE EXTERNAL CATALOG es_logs
PROPERTIES (
  "type"          = "es",
  "hosts"         = "http://es1.internal:9200,http://es2.internal:9200",
  "username"      = "starrocks",
  "password"      = "...",
  "ssl"           = "false",
  "nodes_discovery" = "true",
  "enable_keyword_sniff" = "true"
);

활용

  • 인덱스가 자동으로 테이블처럼 노출됨.
  • WHERE 조건 일부(=, IN, BETWEEN 등)는 ES Query DSL로 푸시다운.
  • 다른 카탈로그(Hive, MySQL)와 JOIN 가능 — 로그 ↔ 사용자 마스터 결합 같은 시나리오.

주의사항

  • ES의 큰 GROUP BY를 그대로 던지면 ES가 OOM 위험. 무거운 집계는 StarRocks 쪽에서 끝내거나 Async MV로 캐시.
  • nested 필드 / multi-value 필드 매핑은 제약이 있음.
  • 매핑이 동적으로 늘어나는 ES 인덱스는 컬럼 폭발에 유의.
클러스터 적용 노트

로그 검색은 ES, 분석은 StarRocks라는 보편 패턴에서, "ES에 있는 데이터 일부를 SQL로도 본다"는 용도에 적합. 평소 분석 트래픽이 많다면 ES 인덱스 → Hive/Iceberg로 일자별 백업하고, 그쪽을 보는 편이 ES 부하 측면에서 안전.

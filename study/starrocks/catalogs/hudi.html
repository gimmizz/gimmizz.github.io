6-3. Hudi Catalog

📌 한 줄 요약

v2.4부터 Apache Hudi 테이블을 읽을 수 있다. CoW와 MoR 모두 지원하며 메타스토어로는 HMS를 가장 흔하게 사용.

기본 PROPERTIES

CREATE EXTERNAL CATALOG hudi_hms
PROPERTIES (
  "type"                  = "hudi",
  "hive.metastore.type"   = "hive",
  "hive.metastore.uris"   = "thrift://hms.example.com:9083",
  -- HDFS / S3 인증은 Hive와 동일
  "aws.s3.endpoint"       = "https://minio.internal:9000",
  "aws.s3.access_key"     = "...",
  "aws.s3.secret_key"     = "...",
  "aws.s3.enable_path_style_access" = "true"
);

활용 기능

  • 스냅샷·Incremental Query 모두 SELECT 가능 (옵션에 따라).
  • Schema 진화(컬럼 추가) 지원.
  • Hudi Time Travel — Iceberg와 유사 시맨틱.

주의사항

  • MoR 테이블의 컴팩션이 누적되면 머지 비용이 폭증 → Hudi 측 hoodie.compact.inline 정책 점검.
  • v3.x 이전 버전에서는 일부 Hudi 옵션(record-level index, secondary index)이 제한적.
  • StarRocks에서 Hudi 쓰기는 미지원. 적재는 Spark/Flink Hudi에서 수행.
클러스터 적용 노트
  • 환경에 Spark Streaming + Hudi MoR 파이프라인이 이미 있다면 그대로 StarRocks를 읽기 엔진으로 붙일 수 있음.
  • 대시보드 노출 전, MoR 테이블의 컴팩션 주기를 짧게(예: 10분)로 운영하면 StarRocks 읽기가 일관되게 빠름.

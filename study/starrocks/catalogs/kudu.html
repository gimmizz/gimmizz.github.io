6-9. Kudu Catalog

📌 한 줄 요약

Apache Kudu의 마스터에 연결해 Kudu 테이블을 SELECT한다. 변경이 잦은 Kudu의 데이터를 OLAP에 바로 합류시킬 때 유용.

PROPERTIES

CREATE EXTERNAL CATALOG kudu_prod
PROPERTIES (
  "type"               = "kudu",
  "kudu.master"        = "kudu-master1:7051,kudu-master2:7051,kudu-master3:7051",
  "kudu.schema-emulation.enabled" = "true",
  "kudu.schema-emulation.prefix"  = "default."
);

활용

  • Kudu의 PK 기반 테이블을 SELECT — 변경이 즉시 반영된 데이터를 분석 쿼리에서 사용.
  • Hive/Iceberg와 JOIN해 마스터(Kudu) ↔ 이벤트(Hive) 결합 분석.

주의사항

  • StarRocks에서 Kudu 쓰기는 미지원.
  • Kudu Tablet Server에 모든 BE가 접근 가능해야 함 (방화벽 확인).
클러스터 적용 노트

사내에 Kudu 기반 실시간 적재가 이미 있다면 그대로 분석에 합류시킬 수 있음. 다만 Kudu의 부하가 걱정이라면 야간에 Hive/Iceberg로 스냅샷 백업하고 그쪽을 분석하는 패턴이 더 안전.

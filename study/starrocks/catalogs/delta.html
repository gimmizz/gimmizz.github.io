6-4. Delta Lake Catalog

📌 한 줄 요약

v2.5부터 Delta Lake 테이블을 직접 SELECT할 수 있다. _delta_log를 직접 읽어 정확한 스냅샷 뷰를 제공.

기본 PROPERTIES

CREATE EXTERNAL CATALOG delta_hms
PROPERTIES (
  "type"                = "deltalake",
  "hive.metastore.uris" = "thrift://hms.example.com:9083",
  "aws.s3.endpoint"     = "https://minio.internal:9000",
  "aws.s3.access_key"   = "...",
  "aws.s3.secret_key"   = "...",
  "aws.s3.enable_path_style_access" = "true"
);

-- AWS Glue로 Delta 테이블을 관리하는 경우
CREATE EXTERNAL CATALOG delta_glue
PROPERTIES (
  "type" = "deltalake",
  "hive.metastore.type" = "glue",
  "aws.glue.region" = "ap-northeast-2",
  "aws.glue.use_instance_profile" = "true"
);

활용 기능

  • Time travel: FOR VERSION AS OF n, FOR TIMESTAMP AS OF '...'
  • Deletion Vector: v3.x에서 단계적 지원. 사용 버전 릴리스 노트 참고.
  • Column Mapping (id/name): v3.x 지원.
  • 스키마 진화: 컬럼 추가 안전.

주의사항

  • StarRocks의 Delta 쓰기는 미지원 — Spark Delta로 적재.
  • _delta_log가 손상되면 카탈로그 전체 쿼리 실패 — Delta 측 무결성 보호 정책 필수.
  • 큰 체크포인트 파일이 있으면 메타 로딩이 느릴 수 있음 — 파티션이 많은 테이블은 첫 쿼리 워밍업 권장.
클러스터 적용 노트
  • Databricks·Spark Delta 파이프라인이 이미 있는 팀이라면 StarRocks를 BI 가속기로 붙이는 패턴이 자연스러움.
  • Iceberg/Delta가 한 HMS에 섞여 있다면 Unified Catalog를 먼저 검토.

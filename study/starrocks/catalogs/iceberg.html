6-2. Iceberg Catalog

📌 한 줄 요약

StarRocks는 Iceberg를 v2.4부터 읽기, v3.x부터 쓰기까지 지원한다. 카탈로그 종류가 다양해서(hive, rest, glue, jdbc, hadoop) 환경에 맞게 선택한다.

카탈로그 종류별 PROPERTIES

HMS 기반

CREATE EXTERNAL CATALOG ic_hms
PROPERTIES (
  "type"                 = "iceberg",
  "iceberg.catalog.type" = "hive",
  "hive.metastore.uris"  = "thrift://hms.example.com:9083"
);

REST Catalog (Tabular, Lakekeeper, Polaris ...)

CREATE EXTERNAL CATALOG ic_rest
PROPERTIES (
  "type"                       = "iceberg",
  "iceberg.catalog.type"       = "rest",
  "iceberg.catalog.uri"        = "https://rest.lakehouse.example.com",
  "iceberg.catalog.warehouse"  = "s3://warehouse/",
  "iceberg.catalog.credential" = "<client-id:client-secret>"
);

AWS Glue 기반

CREATE EXTERNAL CATALOG ic_glue
PROPERTIES (
  "type"                          = "iceberg",
  "iceberg.catalog.type"          = "glue",
  "aws.glue.region"               = "ap-northeast-2",
  "aws.glue.use_instance_profile" = "true"
);

Hadoop / JDBC

# Hadoop (warehouse 경로 직접)
"iceberg.catalog.type"      = "hadoop",
"iceberg.catalog.warehouse" = "hdfs://nn/warehouse"

# JDBC catalog
"iceberg.catalog.type"      = "jdbc",
"iceberg.catalog.uri"       = "jdbc:postgresql://pg/icebergcat",
"iceberg.catalog.jdbc.user" = "iceberg",
"iceberg.catalog.jdbc.password" = "..."

Iceberg에서 활용 가능한 기능

  • Time travel: 
    SELECT * FROM ic_hms.db.tbl FOR VERSION AS OF 8123456789012345678;
SELECT * FROM ic_hms.db.tbl FOR TIMESTAMP AS OF '2026-05-01 00:00:00';
  • 스냅샷·메타 시스템 테이블: $snapshots, $history, $manifests, $partitions, $files, $refs 등을 SELECT 가능.
  • 쓰기 (v3.x): INSERT INTO, INSERT OVERWRITE, CREATE TABLE, ALTER TABLE ADD COLUMN, DELETE(일부), 파티션 진화.
  • Equality / Position Delete 파일 모두 처리.
  • 스키마 진화: 컬럼 추가·이름 변경·타입 위젠. Iceberg가 ID 기반이라 안전.

스토리지 인증

Hive Catalog와 동일. Kerberos, S3/MinIO 액세스 키, Glue Instance Profile 등.

주의사항

  • Delete 파일이 많으면 BE 메모리 부담 ↑ — 정기 컴팩션·rewrite_data_files 권장.
  • REST Catalog 사용 시 인증 토큰 만료에 주의.
  • v3.x의 INSERT는 빠르게 발전 중 — 사용 버전 릴리스 노트 확인.
클러스터 적용 노트
  • 사내 Iceberg가 HMS 기반이라면 iceberg.catalog.type=hive로 시작 — Hive Catalog와 코드 패턴이 거의 같아서 운영 부담이 적음.
  • S3/MinIO 위에 Iceberg를 새로 도입한다면 REST 카탈로그(Lakekeeper, Polaris) 검토 — 다중 엔진 공유에 유리.
  • Time Travel 권한은 별도 역할로 한정 — 잘못된 시점 조회로 분석이 오염되는 사고 방지.

Unified Catalog (v3.2+)

📌 한 줄 요약

한 메타스토어 + 한 스토리지에 Hive · Iceberg · Hudi · Delta · Paimon · Kudu 형식이 섞여 있어도 카탈로그 하나로 묶어 다룰 수 있다. StarRocks가 테이블 형식을 자동 인식해 적절한 리더를 사용.

지원 형식

  • Apache Hive
  • Apache Iceberg
  • Apache Hudi
  • Delta Lake
  • Apache Paimon
  • Apache Kudu
⚠️ 핵심 제약

한 Unified Catalog는 단일 메타스토어 + 단일 스토리지 시스템만 통합한다. 별도 HMS 또는 별도 스토리지(예: HDFS와 S3 동시)를 묶고 싶다면 카탈로그를 분리해야 한다.

CREATE EXTERNAL CATALOG 구문

CREATE EXTERNAL CATALOG <catalog_name>
[COMMENT <comment>]
PROPERTIES (
  "type" = "unified",
  -- MetastoreParams
  -- StorageCredentialParams
  -- MetadataUpdateParams
  -- PaimonCatalogParams (Paimon 사용 시)
  -- KuduCatalogParams (Kudu 사용 시)
);

이름은 문자/숫자/언더스코어 가능, 문자로 시작, 1023자 이내, 대소문자 구분.

MetastoreParams — 메타스토어 옵션

Hive Metastore (HMS)

"unified.metastore.type" = "hive",
"hive.metastore.uris" = "thrift://hms1:9083,thrift://hms2:9083"
-- HA: 콤마로 다중 URI

AWS Glue (S3 전용)

-- Instance Profile
"unified.metastore.type" = "glue",
"aws.glue.use_instance_profile" = "true",
"aws.glue.region" = "ap-northeast-2"

-- Assumed Role
"aws.glue.use_instance_profile" = "true",
"aws.glue.iam_role_arn" = "arn:aws:iam::...:role/sr-glue"

-- IAM User
"aws.glue.use_instance_profile" = "false",
"aws.glue.access_key" = "...",
"aws.glue.secret_key" = "..."

MetadataUpdateParams — 메타데이터 캐시

파라미터기본의미
enable_metastore_cachetrue메타스토어 캐싱 활성
enable_remote_file_cachetrue원격 파일 메타 캐싱
metastore_cache_refresh_interval_sec7200메타스토어 갱신 주기 (초)
remote_file_cache_refresh_interval_sec60파일 메타 갱신 주기
metastore_cache_ttl_sec86400메타 캐시 TTL (24h)
remote_file_cache_ttl_sec129600파일 메타 TTL (36h)

자동 포맷 라우팅 동작

graph LR SQL["SELECT ... FROM unified.db.tbl"] FE["FE — 메타 분석"] HMS["HMS / Glue"] H["Hive Reader"] I["Iceberg Reader"] HU["Hudi Reader"] D["Delta Reader"] P["Paimon Reader"] K["Kudu Reader"] SQL --> FE --> HMS FE -- "table_type=ICEBERG" --> I FE -- "table_type=HUDI" --> HU FE -- "table_type=DELTA" --> D FE -- "Paimon 메타" --> P FE -- "Kudu master 매핑" --> K FE -- "그 외(Hive parquet/orc/text)" --> H

HMS의 테이블 프로퍼티(예: table_type=ICEBERG) 또는 메타 시그니처를 보고 형식을 결정한다. 따라서 HMS의 메타 정합성이 매우 중요.

-- Hive 테이블 생성
CREATE TABLE unified.dwh.summary (
  channel STRING, cnt BIGINT, dt DATE
)
ENGINE = hive
PARTITION BY (dt);

-- Iceberg 테이블 생성
CREATE TABLE unified.dwh.events_iceberg (
  event_time TIMESTAMP, user_id BIGINT, event_type STRING, dt DATE
)
ENGINE = iceberg
PARTITION BY (dt);

-- INSERT (Parquet만)
INSERT INTO unified.dwh.summary SELECT channel, count(*), '2026-05-07' FROM ...;

권한 (RBAC)

GRANT USAGE ON CATALOG unified TO ROLE bi_reader;
GRANT SELECT ON ALL TABLES IN ALL DATABASES IN CATALOG unified TO ROLE bi_reader;

GRANT CREATE DATABASE ON CATALOG unified TO ROLE etl_writer;
GRANT CREATE TABLE ON DATABASE unified.dwh TO ROLE etl_writer;
GRANT INSERT ON TABLE unified.dwh.events_iceberg TO ROLE etl_writer;

주의사항

  • 단일 스토리지·메타스토어 제약: 다른 HMS/스토리지를 추가하려면 별도 카탈로그.
  • 모든 테이블이 같은 자격증명 공유: 권한 분리가 필요한 데이터(PII 등)는 별도 카탈로그.
  • HMS 메타 정합성: table_type 프로퍼티가 잘못되면 라우팅 실패.
  • 형식별 고급 기능(Iceberg Time Travel, Hudi Incremental Query 등)은 Unified를 통해도 그대로 사용 가능.
  • 쓰기는 Hive/Iceberg + Parquet만. 그 외는 원천 시스템에서 적재.
  • Beta/안정화 단계: 프로덕션 적용 전 PoC 필수.
클러스터 적용 노트
  • HMS 안에 Hive parquet과 Iceberg가 섞여 있는 게 일반적이라면 처음부터 Unified로 시작이 깔끔. BI/분석가에게도 카탈로그 이름이 하나로 보여 혼란이 적음.
  • 같은 HMS·스토리지지만 권한 정책이 다른 데이터셋(PII 등)은 별도 카탈로그로 분리.
  • BE에 Data Cache 활성 + 1개 SSD 전용으로 두면 외부 카탈로그 쿼리가 내부 테이블 수준 응답속도. (Hive 페이지의 Data Cache 섹션 참고)
  • HMS 메타 캐시 갱신 주기(7200초)는 운영에서 너무 길면 새 파티션 인식이 늦음 — 적재 직후 REFRESH EXTERNAL TABLE를 ETL 끝에 명시적으로 실행하는 패턴이 안전.
  • v3.2 도입 후 빠르게 발전 중인 영역 — 사용 버전 릴리스 노트의 Behavior change 항목을 매번 확인.

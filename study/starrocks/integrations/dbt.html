INT-2. dbt-starrocks 어댑터

dbt에서 StarRocks를 타깃 DB로 사용. 공식 어댑터 dbt-starrocks (PyPI) 또는 GitHub StarRocks/dbt-starrocks.

설치

pip install dbt-core dbt-starrocks

profiles.yml

my_project:
  target: dev
  outputs:
    dev:
      type: starrocks
      host: fe1.example.com
      port: 9030
      schema: mart            # database 이름
      username: dbt_user
      password: "{{ env_var('SR_PASSWORD') }}"
      catalog: default_catalog
      threads: 8
      keepalives_idle: 0
      retries: 1
    prod:
      type: starrocks
      host: lb.example.com
      port: 9030
      schema: mart
      username: dbt_user
      password: "{{ env_var('SR_PASSWORD_PROD') }}"
      catalog: default_catalog
      threads: 16

지원 머티리얼라이제이션

dbtSR 매핑
viewVIEW
table일반 테이블 (CTAS)
incrementalPRIMARY KEY 테이블 + INSERT/UPSERT
materialized_viewAsync MV

model 예 — table

-- models/mart/orders_daily.sql
{{ config(
    materialized='table',
    properties={
      'replication_num': '3',
      'compression': 'ZSTD'
    },
    partition_by=["dt"],
    distributed_by=["user_id"],
    unique_key='order_id'
) }}

SELECT
  date_trunc('day', updated_at) AS dt,
  order_id, user_id, status, amount
FROM {{ source('cdc', 'orders_raw') }}

model 예 — incremental (PK)

{{ config(
    materialized='incremental',
    unique_key='order_id',
    table_type='primary_key',
    distributed_by=["order_id"]
) }}

SELECT *
FROM {{ source('cdc', 'orders_raw') }}
{% if is_incremental() %}
WHERE updated_at > (SELECT max(updated_at) FROM {{ this }})
{% endif %}

model 예 — Materialized View

{{ config(
    materialized='materialized_view',
    refresh_method='ASYNC',
    refresh_schedule="EVERY (INTERVAL 1 HOUR)",
    partition_by=["dt"],
    distributed_by=["user_id"]
) }}

SELECT date_trunc('day', updated_at) AS dt,
       user_id, count(*) cnt, sum(amount) amt
FROM {{ ref('orders_daily') }}
GROUP BY 1, 2

실행

dbt run -s mart.orders_daily
dbt test
dbt docs generate && dbt docs serve

External Catalog 모델

schema를 hive_prod.dwh처럼 catalog.database 형태로 두면 외부 카탈로그를 타깃으로 빌드할 수 있다(어댑터 버전에 따라 동작 다름 — 확인 후 사용).

클러스터 적용 노트
  • dbt_user 전용 계정에 USAGE/SELECT/INSERT/CREATE만 부여. DROP DATABASE 권한은 위험.
  • incremental 모델은 PK 테이블로 운영. 백필은 --full-refresh를 신중히.
  • dbt run을 Airflow/Dagster로 묶으면 다음 페이지(INT-3) 패턴.

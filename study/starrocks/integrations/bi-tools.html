INT-1. BI 도구 연결 (Tableau · Superset · Metabase · Redash)

StarRocks는 MySQL 와이어 프로토콜이라 "MySQL"로 연결하면 거의 모든 BI 도구가 즉시 동작합니다. 포트는 9030, 사용자 비밀번호는 SR 사용자 그대로.

Tableau

  1. 커넥터: "MySQL" 선택 (또는 "JDBC" + MySQL Connector/J).
  2. 호스트: fe1 (또는 LB), 포트: 9030, 사용자: SR 계정.
  3. 드라이버 경로 안내가 뜨면 MySQL Connector/J 다운로드 후 지정.
  4. (선택) 드라이버 옵션: useSSL=false&allowPublicKeyRetrieval=true

Apache Superset

pip install mysqlclient
# 또는
pip install starrocks   # 공식 SR Superset 다이얼렉트(있는 버전 기준)

# Superset Database 등록 시 SQLAlchemy URI:
mysql://user:password@fe1:9030/default_catalog
# 또는 SR 다이얼렉트 사용 시
starrocks://user:password@fe1:9030/default_catalog

Metabase

  1. "Add a database" → "MySQL" 선택.
  2. 호스트/포트/사용자/DB 입력. DB 이름은 default_catalog.<database> 또는 <database>.

Redash

Data Source: "MySQL" (또는 RDS MySQL). 호스트/포트/사용자/암호.

DBeaver / DataGrip (분석 IDE)

  • MySQL 드라이버 그대로 사용. Driver Properties에 useSSL=false 추가.
  • 또는 StarRocks 전용 드라이버 옵션을 추가하면 더 정확한 메타 표시.

베스트 프랙티스

  • BI에는 읽기 전용 계정만 — RBAC로 mart DB SELECT만.
  • BI 트래픽은 rg_bi Resource Group에 묶고 concurrency_limit 충분히.
  • FE LB 뒤에 두면 한 FE 다운에도 무중단.
  • Tableau Extracts는 SR이 충분히 빠르므로 거의 불필요. Live Query 권장.

External Catalog를 BI에서

BI 도구의 DB 선택에서 다른 카탈로그도 보고 싶다면 사용자에게 카탈로그 USAGE 권한 부여:

GRANT USAGE ON CATALOG hive_prod TO ROLE bi_reader;
GRANT SELECT ON ALL TABLES IN ALL DATABASES IN CATALOG hive_prod TO ROLE bi_reader;

각 BI 도구에서 catalog는 보통 "스키마"의 상위처럼 보이거나 기본 SQLAlchemy URI에 명시.

클러스터 적용 노트
  • BI 트래픽은 Observer FE를 1~2대 추가하면 읽기 분산에 효과적.
  • 이미 만든 Async MV로 BI SQL이 자동 가속되도록 분석가가 모르는 사이에 응답이 빨라지는 패턴이 가장 좋음.
  • 대시보드 응답 SLA 위반은 보통 (1) 통계 stale, (2) Resource Group 미분리, (3) JOIN 셔플 폭주 — 이 셋부터 점검.

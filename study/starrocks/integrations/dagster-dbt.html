INT-3. Dagster + dbt-starrocks 파이프라인

📌 한 줄 요약

Dagster는 dbt 모델을 자동으로 asset으로 인식한다. dagster-dbt로 dbt-starrocks 프로젝트를 감싸면, 적재→변환→MV 트리거를 한 그래프로 관리할 수 있다.

아키텍처

설치

pip install dagster dagster-webserver dagster-dbt dbt-core dbt-starrocks

프로젝트 구조

my_pipeline/
├── pyproject.toml
├── my_pipeline/
│   ├── __init__.py
│   ├── definitions.py        # Dagster Definitions
│   └── dbt/
│       └── starrocks_proj/   # dbt 프로젝트
│           ├── dbt_project.yml
│           ├── models/
│           └── profiles.yml

definitions.py

from pathlib import Path
from dagster import Definitions, ScheduleDefinition, define_asset_job
from dagster_dbt import DbtCliResource, dbt_assets, DbtProject

DBT_DIR = Path(__file__).parent / "dbt" / "starrocks_proj"
dbt_project = DbtProject(project_dir=DBT_DIR)
dbt_project.prepare_if_dev()

@dbt_assets(manifest=dbt_project.manifest_path)
def starrocks_dbt_assets(context, dbt: DbtCliResource):
    yield from dbt.cli(["build"], context=context).stream()

dbt_resource = DbtCliResource(project_dir=str(DBT_DIR), profiles_dir=str(DBT_DIR))

daily_job = define_asset_job("daily_job", selection="*")
schedule  = ScheduleDefinition(job=daily_job, cron_schedule="30 2 * * *")  # 매일 02:30

defs = Definitions(
    assets=[starrocks_dbt_assets],
    resources={"dbt": dbt_resource},
    schedules=[schedule],
)

실행

dagster dev   # 개발 모드 — 웹 UI http://localhost:3000

# 운영 배포
dagster-webserver -h 0.0.0.0 -p 3000 -m my_pipeline.definitions
dagster-daemon run -m my_pipeline.definitions

MV refresh 트리거

dbt가 마트 테이블을 다 만든 직후 SR의 MV를 명시적으로 리프레시하고 싶다면, dbt 모델 사이에 SR REFRESH MATERIALIZED VIEW를 호출하는 Dagster asset을 추가하거나 dbt의 post-hook으로 처리.

# dbt 모델의 post-hook 예
{{ config(post_hook="REFRESH MATERIALIZED VIEW mart.mv_orders_d FORCE") }}

-- 또는 별도 dagster op
from dagster import op
import pymysql
@op
def refresh_mv(context):
    conn = pymysql.connect(host="fe1", port=9030, user="dbt_user",
                           password="...", autocommit=True)
    with conn.cursor() as cur:
        cur.execute("REFRESH MATERIALIZED VIEW mart.mv_orders_d FORCE")

Sensor — 외부 적재 완료를 감지

from dagster import sensor, RunRequest, SensorEvaluationContext
@sensor(job=daily_job)
def kafka_done_sensor(context: SensorEvaluationContext):
    # 예: SR에서 max(updated_at)이 오늘로 도달하면 dbt 실행
    import pymysql
    conn = pymysql.connect(host="fe1", port=9030, user="dbt_user", password="...")
    with conn.cursor() as cur:
        cur.execute("SELECT max(updated_at) FROM mart.raw_orders")
        ts = cur.fetchone()[0]
    if ts and ts.date().isoformat() == context.cursor:
        return  # 이미 처리
    return RunRequest(run_key=str(ts), run_config={})

관측 / UI

  • Dagster UI에서 asset graph로 dbt 모델 의존성을 시각화.
  • 각 모델 실행 시간/실패 그래프 자동.
  • SR 측의 information_schema.task_runs(MV)와 dagster의 jobs를 함께 보면 파이프라인 끝점이 분명.
클러스터 적용 노트
  • 전용 계정 분리: dbt_user, dagster_user를 분리하고 RBAC로 권한 최소화.
  • Resource Group 격리: dbt run이 BI 트래픽을 잡아먹지 않게 rg_etl 사용.
  • 장애 대응: Routine Load PAUSED → Dagster sensor가 잡아 Slack/웍스 알림.
  • 본 페이지의 코드는 예시 패턴이며 dagster/dbt 어댑터 버전에 따라 미세 차이가 있으니 운영 전 검증 필수.

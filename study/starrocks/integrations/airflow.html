INT-4. Airflow 연동 패턴

Connection 등록

Airflow Web UI → Admin → Connections.

  • Conn Id: starrocks_default
  • Conn Type: MySQL (또는 generic)
  • Host/Port: fe1 / 9030
  • Login/Password: SR 계정
  • Extra: {"charset":"utf8mb4"}

MySqlOperator로 SQL 실행

from airflow import DAG
from airflow.providers.mysql.operators.mysql import MySqlOperator
from datetime import datetime

with DAG("sr_daily", start_date=datetime(2026,5,1), schedule="0 2 * * *", catchup=False) as dag:
    rebuild = MySqlOperator(
        task_id="rebuild_orders_d",
        mysql_conn_id="starrocks_default",
        sql="""
            INSERT OVERWRITE mart.orders_d PARTITION (dt='{{ ds }}')
            SELECT date_trunc('day', updated_at) dt, user_id, sum(amount) amt
            FROM mart.orders WHERE updated_at::date='{{ ds }}'
            GROUP BY 1, 2;
        """
    )
    refresh_mv = MySqlOperator(
        task_id="refresh_mv",
        mysql_conn_id="starrocks_default",
        sql="REFRESH MATERIALIZED VIEW mart.mv_orders_d FORCE;"
    )
    rebuild >> refresh_mv

Stream Load Operator 패턴

from airflow.providers.http.operators.http import SimpleHttpOperator

stream_load = SimpleHttpOperator(
    task_id="load_orders",
    http_conn_id="starrocks_http",      # http://fe1:8030
    endpoint="/api/mart/orders/_stream_load",
    method="PUT",
    headers={
      "Authorization": "Basic ...",      # base64(user:pass)
      "label": "orders_{{ ts_nodash }}",
      "format": "CSV",
      "column_separator": ",",
      "columns": "order_id,user_id,status,amount,updated_at",
    },
    data=open("/data/orders.csv","rb").read(),
    log_response=True,
)

StarRocks Provider

SR 공식/커뮤니티 Airflow provider가 있는 경우 apache-airflow-providers-starrocks를 설치해 StarRocksOperator / StarRocksStreamLoadOperator 등을 사용. 없는 환경이라면 위 MySqlOperator/HttpOperator로 충분.

Airflow + dbt

BashOperator 또는 airflow-dbt / cosmos로 dbt run을 실행. 일반적으로 Dagster + dbt가 dbt 그래프 시각화에 더 강하지만, 팀 표준이 Airflow라면 그대로 사용 OK.

dbt_run = BashOperator(
  task_id="dbt_run",
  bash_command="cd /opt/dbt/proj && dbt run --target prod",
)

SQL Sensor

from airflow.providers.mysql.sensors.sql import SqlSensor
SqlSensor(
  task_id="wait_data",
  conn_id="starrocks_default",
  sql="SELECT count(*) FROM mart.orders WHERE updated_at::date='{{ ds }}'",
  poke_interval=60, timeout=3600
)
클러스터 적용 노트
  • BI/ETL 분리: Airflow의 SR 사용은 rg_etl Resource Group으로.
  • Stream Load의 label은 항상 unique — Airflow의 {{ ts_nodash }}를 활용해 retry 안전.
  • 큰 백필은 한 DAG run 안에서 파티션 단위로 분할(Dynamic Task Mapping) — 한 SQL로 모두 OVERWRITE는 위험.

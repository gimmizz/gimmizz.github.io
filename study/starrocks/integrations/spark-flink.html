INT-5. Spark / Flink Connector

Spark Connector

StarRocks 공식 Spark Connector — 읽기·쓰기 모두 지원. Maven 좌표는 버전에 따라 다름. com.starrocks:starrocks-spark-connector-3.x_2.12 형태.

읽기

val df = spark.read.format("starrocks")
  .option("starrocks.fenodes", "fe1:8030,fe2:8030")
  .option("starrocks.table.identifier", "mart.orders")
  .option("starrocks.user", "etl")
  .option("starrocks.password", "...")
  .load()
df.show()

쓰기 (Stream Load 기반)

df.write.format("starrocks")
  .option("starrocks.fenodes", "fe1:8030,fe2:8030")
  .option("starrocks.table.identifier", "mart.orders")
  .option("starrocks.user", "etl")
  .option("starrocks.password", "...")
  .mode("append")
  .save()

Flink Connector

스트리밍 적재 표준. Flink CDC + StarRocks Connector로 OLTP → OLAP 실시간 ingestion.

SQL 모드 (Flink SQL)

CREATE TABLE sr_orders (
  order_id   BIGINT,
  user_id    BIGINT,
  status     STRING,
  amount     DECIMAL(18,2),
  updated_at TIMESTAMP(3)
) WITH (
  'connector' = 'starrocks',
  'jdbc-url'  = 'jdbc:mysql://fe1:9030',
  'load-url'  = 'fe1:8030;fe2:8030',
  'database-name' = 'mart',
  'table-name'    = 'orders',
  'username' = 'etl',
  'password' = '...',
  'sink.properties.format' = 'json',
  'sink.properties.strip_outer_array' = 'true'
);

INSERT INTO sr_orders
SELECT order_id, user_id, status, amount, updated_at
FROM kafka_orders;

DataStream 모드

StarRocksSinkOptions options = StarRocksSinkOptions.builder()
  .withProperty("jdbc-url", "jdbc:mysql://fe1:9030")
  .withProperty("load-url", "fe1:8030;fe2:8030")
  .withProperty("database-name","mart")
  .withProperty("table-name","orders")
  .withProperty("username","etl")
  .withProperty("password","...")
  .withProperty("sink.properties.format", "json")
  .build();

stream.addSink(StarRocksSink.sink(options));

Flink CDC + SR 패턴

-- 소스: Flink CDC MySQL
CREATE TABLE src_orders (...) WITH ('connector'='mysql-cdc', ...);

-- 싱크: StarRocks PK 테이블 (자동 UPSERT/DELETE)
CREATE TABLE snk_orders (...) WITH ('connector'='starrocks', ...);

INSERT INTO snk_orders SELECT * FROM src_orders;

자주 쓰는 옵션

  • sink.buffer-flush.max-rows / max-bytes / interval-ms — 배치 사이즈/시간.
  • sink.properties.format — json / csv.
  • sink.properties.partial_update — PK 테이블의 일부 컬럼만 갱신.
  • sink.label-prefix — Stream Load label 접두어. 충돌 방지.
클러스터 적용 노트
  • 실시간 ingestion은 Flink + SR 패턴이 가장 표준. Spark는 주로 일배치 / 백필.
  • label 접두어를 잡 이름으로 두면 SR SHOW LOAD에서 추적이 쉬움.
  • Spark 큰 INSERT는 Resource Group으로 격리. 야간에 실행.

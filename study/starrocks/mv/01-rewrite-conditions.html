7-1. MV Query Rewrite — 무엇이 자동으로 바뀌나

📌 한 줄 요약

옵티마이저는 사용자 SQL과 MV 정의를 비교해 자동 치환을 시도한다. 일치(exact) 또는 유도(derivable) 두 종류가 있다.

활성 조건

  1. 세션 변수: SET enable_materialized_view_rewrite = true; (기본 true)
  2. MV 속성: "enable_query_rewrite" = "true"
  3. MV 상태: ACTIVE (가장 최근 리프레시가 성공)
  4. 베이스 테이블 데이터가 MV 리프레시 이후 의미 있게 변하지 않았거나, MV가 그 변경을 흡수할 수 있는 상태(파티션 정합)

exact vs derivable

graph LR Q1["SELECT dt, count(*) GROUP BY dt"] MV1["MV: SELECT dt, count(*) GROUP BY dt"] Q1 -- "exact match" --> MV1 Q2["SELECT dt, sum(amt) GROUP BY dt"] MV2["MV: SELECT dt, channel, sum(amt) GROUP BY dt, channel"] Q2 -- "derivable
(dt, channel) → (dt) 가능" --> MV2

잘 동작하는 패턴

  • 같은 GROUP BY 또는 더 거친 GROUP BY (MV가 더 세밀)
  • WHERE가 MV의 WHERE를 포함 또는 같음 (MV가 더 광범위한 데이터를 가진다)
  • JOIN은 MV에 같은 테이블이 같은 키로 정의되어 있어야 함
  • 윈도 함수는 일반적으로 rewrite 안 함 — pre-aggregate 형태로만
  • 외부 카탈로그(Hive/Iceberg)도 동일하게 rewrite 가능 (v3.x)

잘 안 되는 패턴

  • MV의 WHERE이 사용자 쿼리보다 더 좁음 (필요한 데이터를 못 가짐)
  • JOIN 종류가 다름 (LEFT vs INNER 등)
  • 비결정적 함수 (random, current_user 등)
  • HAVING/QUALIFY가 MV에 미반영
  • MV 자체가 stale (베이스 변경을 못 따라잡음)

EXPLAIN으로 확인

EXPLAIN SELECT dt, count(*) FROM events GROUP BY dt;
-- 출력의 SCAN 단에 "RewriteByMV: mv_event_d" 같은 표시.
-- 또는 v3.x의 EXPLAIN의 "MaterializedView" 섹션.

-- 강제로 끔
SELECT /*+ MV_REWRITE_OFF */ ... ;

집계 롤업

MV가 더 세밀한 GROUP BY를 가지면, 더 거친 쿼리는 MV에서 추가 GROUP BY로 계산 가능. SUM/COUNT/MAX/MIN은 자유롭게 롤업되지만, AVG는 보통 SUM/COUNT로 분해해야 함.

-- 권장: AVG를 직접 저장하지 말고 sum, cnt를 저장
CREATE MATERIALIZED VIEW mv_amt
AS SELECT dt, channel, sum(amt) sum_amt, count(*) cnt
FROM events GROUP BY dt, channel;

-- 사용자 쿼리는 이렇게 쓰면 자동 rewrite + AVG 계산 가능
SELECT dt, sum(sum_amt)/sum(cnt) AS avg_amt FROM ... ;

외부 카탈로그 MV (v3.x)

CREATE MATERIALIZED VIEW mv_hive_d
PARTITION BY dt DISTRIBUTED BY HASH(user_id)
REFRESH ASYNC EVERY (INTERVAL 1 HOUR)
PROPERTIES ("enable_query_rewrite"="true", "force_external_table_query_rewrite"="true")
AS SELECT date_trunc('day', event_time) dt, user_id, count(*) cnt
   FROM hive_prod.dwh.events
   GROUP BY 1, 2;
클러스터 적용 노트
  • MV 후보를 결정할 땐 일정 기간(1주)의 audit log에서 가장 자주 도는 SQL Top-N을 추출 → MV 정의로 변환.
  • Rewrite가 안 되는 이유는 보통 (1) stale, (2) WHERE 좁음, (3) JOIN 종류 불일치 셋 중 하나.
  • 운영에서 enable_query_rewrite를 끄지 마세요 — 기본 활성이 안전.

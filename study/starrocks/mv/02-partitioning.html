7-2. MV 파티션 정합 (Partition Alignment)

📌 한 줄 요약

MV의 파티션을 베이스 테이블의 파티션에 맞추면 (1) Incremental Refresh가 가능하고 (2) Rewrite가 stale 파티션만 우회한다. 베이스 일자 파티션이라면 MV도 일자 파티션.

베이스와 같은 키로 PARTITION BY 정의

CREATE MATERIALIZED VIEW mv_event_user_d
PARTITION BY dt                 -- 베이스의 파티션 키와 일치
DISTRIBUTED BY HASH(user_id)
REFRESH ASYNC EVERY (INTERVAL 1 HOUR)
AS
SELECT date_trunc('day', event_time) AS dt,
       user_id,
       count(*) AS cnt
FROM events
GROUP BY 1, 2;

여기서 events는 PARTITION BY date_trunc('day', event_time)로 정의되어 있고, MV의 dt는 그것과 같은 표현식이라 옵티마이저가 1:1 매핑을 인식.

Incremental Refresh — 변경된 파티션만

-- 베이스의 dt='2026-05-07' 파티션이 변경되면, MV의 같은 dt 파티션만 다시 만든다.
REFRESH MATERIALIZED VIEW mv_event_user_d
        PARTITION START('2026-05-07') END('2026-05-08');

관련 PROPERTIES

키의미
partition_refresh_number한 번에 리프레시할 파티션 수 (기본 1)
excluded_trigger_tables이 테이블 변경은 MV 갱신을 트리거하지 않음 (예: 차원 테이블)
auto_refresh_partitions_limit자동 리프레시 시 최근 N개 파티션만
partition_ttl_numberMV의 파티션 보존 개수 (오래된 자동 삭제)

다중 베이스 테이블 + 파티션 정합

여러 테이블을 JOIN하는 MV에서 파티션 정합은 대표(파티션 reference) 테이블 하나를 기준으로 한다. 보통 가장 큰 사실(fact) 테이블이 파티션 키 제공자.

CREATE MATERIALIZED VIEW mv_join_d
PARTITION BY dt
DISTRIBUTED BY HASH(user_id)
REFRESH ASYNC EVERY (INTERVAL 1 HOUR)
AS
SELECT
  date_trunc('day', e.event_time) dt,
  e.user_id, u.grade, count(*) cnt
FROM events e            -- 파티션 reference (큰 fact)
LEFT JOIN dim_user u ON e.user_id = u.user_id
GROUP BY 1, 2, 3;

이때 dim_user는 작은 차원 테이블이라 거의 변하지 않음 → excluded_trigger_tables에 추가하면 dim_user 변경으로 MV가 통째로 리프레시되는 사고를 막을 수 있다.

외부 카탈로그 베이스

Hive/Iceberg 베이스의 MV도 파티션 정합 가능. 단 메타 캐시가 stale이면 새 파티션을 인식 못 할 수 있어 enable_background_refresh_connector_metadata·REFRESH CATALOG를 함께 운영.

상태 점검

SHOW MATERIALIZED VIEW mv_event_user_d\G
SHOW PARTITIONS FROM mv_event_user_d;
-- 각 파티션의 isCacheValid / lastRefreshTime 등으로 정합성 확인.
클러스터 적용 노트
  • 파티션 정합은 MV 운영의 핵심. 풀 리프레시는 약 17 BE 환경에서도 곧 비싸짐. 처음부터 Incremental 가능한 형태로 설계.
  • 차원 테이블은 excluded_trigger_tables로 빼두기 — 차원 변경은 보통 야간에 dim_user 갱신만 하고, MV는 다음 정기 리프레시에서 따라잡음.
  • partition_ttl_number로 MV 파티션 보존 기간을 명시하면 디스크 사용량이 안정적.

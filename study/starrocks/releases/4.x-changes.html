4.x 주요 변경 사항

💡 정확한 변경 내역

이 페이지는 운영 관점에서 영향이 큰 변경을 모았습니다. 전체 릴리스 노트는 공식 release notes를 참고하세요.

4.1

  • 외부 카탈로그 Query Rewrite 안정화 — Hive/Iceberg 베이스 MV가 더 적극적으로 자동 사용.
  • Iceberg/Delta Time Travel 안정화 + Equality Delete 처리 강화.
  • Routine Load 성능/안정성 개선.
  • Shared-data 모드: 캐시 적중률·콜드 스타트 개선.
  • Pipeline 엔진 추가 최적화.

4.0

  • JDK 17 권장 — 이전 JDK 11도 동작은 하나 권장 라인이 17로 이동.
  • Async MV의 Query Rewrite 기본 활성 + 외부 카탈로그 베이스 MV 안정화.
  • Storage Volume 개념 도입(Shared-data 모드 표준화).
  • 인증·인가 RBAC 깊이 강화 (DB·Catalog 단위 권한).
  • Inverted Index / N-Gram Bloom Filter 정식 지원.
  • Routine Load의 backpressure 처리 개선.
  • FQDN 모드 안정화 (K8s 친화).

호환성 깨진 변경 (운영 영향)

  • 일부 시스템 변수 기본값 조정 — 업그레이드 후 SHOW VARIABLES로 차이 검증 권장.
  • JDK 11 이하 환경은 4.x에서 권장 X.
  • 일부 deprecated 함수가 실제로 제거 (예: 일부 구 SQL 함수). 운영 SQL 점검 필요.
  • RBAC 모델 변경으로 v2.x 시절의 일부 권한 부여 방식이 달라짐.

클러스터 적용 노트
  • 새 클러스터는 4.x로 시작이 표준. JDK 17 + 메이저 라인 4.x 고정.
  • 마이너 패치(4.1.x)는 비교적 자주 나오니 분기에 한 번 정도는 검토하고 적용 일정을 잡는 게 좋음.
  • 릴리스 노트의 "Behavior change" 섹션은 매번 정독 — 가장 큰 사고가 여기서 옴.

SQL-8. MySQL 호환성 — 비슷하지만 다른 점

StarRocks는 MySQL 와이어 프로토콜을 그대로 사용해 도구 호환성이 매우 높지만, 분석 워크로드를 위해 설계된 SQL 다이얼렉트이므로 MySQL과는 다음과 같은 차이가 있습니다.

DML

주제MySQLStarRocks
UPDATE/DELETE모든 테이블Primary Key 테이블에서 일반적으로 가능. 다른 모델은 파티션 단위 변경.
트랜잭션BEGIN/COMMIT/ROLLBACK적재 단위(Stream/Routine Load 등)에 트랜잭션 — 다중 문장 트랜잭션은 미지원
INSERT label없음중복 방지 label 강력 권장
FK / CHECK / UNIQUE지원제약 미강제 (모델 자체로 충돌 방지)

DDL

  • 테이블 생성 시 PRIMARY KEY / DUPLICATE KEY / AGGREGATE KEY / UNIQUE KEY 모델을 명시.
  • 파티션·버킷팅·복제본을 거의 항상 함께 정의.
  • 인덱스 종류가 BITMAP / BLOOM / N-GRAM 등 다양 — MySQL B-Tree 인덱스와 의미가 다름.
  • 스토어드 프로시저는 매우 제한적/없음.

데이터 타입

타입MySQL과 차이
VARCHAR최대 1MB까지. 길이 제한 표기 가능.
TEXT / STRINGSTRING은 사실상 매우 큰 VARCHAR.
JSON네이티브 JSON 타입 + 풍부한 함수.
BITMAP / HLLStarRocks 고유 — UV 분석에 핵심.
ARRAY / MAP / STRUCT네이티브 지원.
ENUM / SET없음 — TINYINT/VARCHAR로 대체.
DECIMAL지원. precision/scale 명시.

자주 다른 문법

-- 1) IFNULL/COALESCE 모두 OK. 단 NULL 비교는 =/!= 대신 IS NULL.

-- 2) UPSERT는 INSERT ON DUPLICATE KEY UPDATE 가 아니라
--    Stream Load + PK 테이블 (자동 UPSERT) 또는
--    Primary Key UPDATE 문.

-- 3) LIMIT OFFSET은 OK. 큰 OFFSET은 비싸 — 키 기반 페이지네이션 권장.
SELECT * FROM t WHERE id > ? ORDER BY id LIMIT 100;

-- 4) GROUP BY: SELECT 절 컬럼이 GROUP BY에 모두 있어야 함 (sql_mode='ONLY_FULL_GROUP_BY')

-- 5) 형변환은 CAST 또는 ::TYPE
SELECT '123'::INT, CAST('123' AS INT);

-- 6) 백틱 식별자 가능: `my-col`
-- 7) 더블쿼터는 문자열이 아니라 식별자(표준 SQL). 기본 sql_mode 설정 확인.

쿼리 변수 / SET

SET parallel_fragment_exec_instance_num = 8;
SET enable_materialized_view_rewrite = true;
SET query_mem_limit = 32 * 1024 * 1024 * 1024;

-- 글로벌
SET GLOBAL ...;
-- 자기 자신만
SET SESSION ...;

-- 모두 보기
SHOW VARIABLES LIKE '%mv%';

인증·권한

  • MySQL 사용자처럼 보이지만, RBAC가 더 강력하고 catalog/database/table 단위 GRANT가 다양.
  • GRANT ALL ON *.* 같은 권한은 너무 광범위 — 가능한 한 좁게.
  • LDAP/Kerberos 통합 가능 (5-4 보안).

도구 호환

  • OK: MySQL CLI, JDBC, Python mysql-connector, DBeaver, Tableau, Superset, Metabase, Redash, dbt(어댑터).
  • 주의: MySQL specific 함수(예: 일부 stored procedure, 트리거)나 트랜잭션 가정.
클러스터 적용 노트
  • 분석가 onboarding 자료에 본 페이지를 첫 장으로. "MySQL과 거의 같지만 이 5가지는 다르다"만 알려줘도 사고 방지에 큰 효과.
  • 운영 ETL이 INSERT ON DUPLICATE KEY 같은 MySQL 패턴에 의존했다면, Stream Load + PK 테이블 패턴으로 표준화.

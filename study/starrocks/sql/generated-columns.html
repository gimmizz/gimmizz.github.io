SQL — Generated Columns

다른 컬럼의 표현식으로 자동 계산되는 컬럼. JSON/날짜 추출 등을 적재 단계에서 컬럼화하면 쿼리·인덱스 가속에 유리.

선언

CREATE TABLE event_log (
  event_id   BIGINT NOT NULL,
  ts         DATETIME NOT NULL,
  payload    JSON,
  -- 표현식 기반 컬럼 (저장됨)
  dt         DATE   AS (CAST(ts AS DATE)),
  user_id    BIGINT AS (get_json_int(payload, '$.user_id')),
  event_type VARCHAR(32) AS (get_json_string(payload, '$.type'))
)
DUPLICATE KEY(event_id, ts)
PARTITION BY dt
DISTRIBUTED BY HASH(user_id)
PROPERTIES ("replication_num" = "3");

적재할 때는 입력 컬럼만 적재하면 된다 — generated 컬럼은 자동 계산.

INSERT INTO event_log (event_id, ts, payload) VALUES
  (1, '2026-05-07 10:00:00', '{"user_id":100,"type":"view"}');
-- dt, user_id, event_type 자동 채워짐

왜 쓰나

  • 파티션 키로 사용 — payload 안의 dt를 추출해 파티션 키로 쓸 수 있다.
  • 인덱스 부착 — generated 컬럼에 Bitmap/Bloom 인덱스를 걸어 가속.
  • 쿼리 단순화 — JSON 추출을 매번 쓰지 않아도 됨.
  • 일관성 — 정의가 스키마에 박혀 있어 모든 쓰기가 같은 규칙.

ALTER로 추가

ALTER TABLE event_log
  ADD COLUMN region VARCHAR(8) AS (get_json_string(payload, '$.region'));

제약

  • generated 컬럼은 읽기 전용: 직접 INSERT/UPDATE 못 함 (입력 컬럼이 결정).
  • 표현식은 결정적(deterministic)이어야 — now(), rand() 등은 사용 불가.
  • 표현식 내부의 입력 컬럼이 ALTER로 바뀌면 재계산이 필요할 수 있음.
  • UDF를 generated 표현식에 쓰려면 추가 검증 필요.
클러스터 적용 노트
  • JSON payload를 받는 이벤트 테이블에서 가장 가치가 큼. 자주 조회되는 키를 generated 컬럼으로 미리 빼두면 쿼리 비용/인덱스 효과가 크게 향상.
  • 같은 효과를 ETL에서 미리 컬럼화해도 OK이지만, generated 컬럼은 ETL 변경 없이 스키마만 손대면 되어 운영 부담이 작음.

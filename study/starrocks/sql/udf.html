SQL — UDF (User-Defined Function)

표준 함수로 처리하기 어려운 변환·검증·암호화 등은 UDF로. StarRocks는 Java UDF(스칼라/UDAF/UDTF) 그리고 v3.x부터 Python UDF(스칼라)를 지원.

Java 스칼라 UDF

// 1) Java 코드
package com.example.sr;
public class AddOne {
  public Integer evaluate(Integer x) {
    if (x == null) return null;
    return x + 1;
  }
}
// 2) Maven으로 jar 빌드 → S3/HDFS/HTTP에 업로드
-- 3) 등록
CREATE FUNCTION add_one(INT) RETURNS INT
PROPERTIES (
  "type"     = "StarrocksJar",
  "file"     = "http://internal/sr-udf/add-one-1.0.jar",
  "symbol"   = "com.example.sr.AddOne",
  "object_file" = "..."
);

SELECT add_one(10);   -- 11

Java UDAF (집계)

public class MyAvg {
  public static class State {
    public long count;
    public double sum;
  }
  public State create()                              { return new State(); }
  public void   destroy(State s)                     {}
  public void   update(State s, Double v)            { if (v != null) { s.count++; s.sum += v; } }
  public void   merge(State s, State other)          { s.count += other.count; s.sum += other.sum; }
  public Double finalize(State s)                    { return s.count == 0 ? null : s.sum / s.count; }
  public void   serialize(State s, java.nio.ByteBuffer buf) { ... }
  public void   deserialize(State s, java.nio.ByteBuffer buf) { ... }
}
CREATE AGGREGATE FUNCTION my_avg(DOUBLE) RETURNS DOUBLE
PROPERTIES (...);
SELECT my_avg(amount) FROM orders;

Java UDTF (테이블 함수)

한 행을 여러 행으로 펼치는 함수. process가 여러 번 emit.

public class Split {
  public Iterator<String> process(String s, String sep) {
    if (s == null) return Collections.emptyIterator();
    return Arrays.asList(s.split(sep)).iterator();
  }
}

Python UDF (스칼라)

CREATE FUNCTION add_one(INT) RETURNS INT
PROPERTIES (
  "type"   = "Python",
  "symbol" = "add_one",
  "input"  = "scalar",
  "file"   = "http://internal/sr-udf/add_one.py"
);

def add_one(x):
    return None if x is None else x + 1

주의

  • 성능 페널티: 네이티브 벡터화에 비해 UDF는 행 단위로 호출 → 수백 배 느릴 수 있음. 큰 데이터에는 가급적 표준 함수.
  • JAR 배포: 모든 BE에서 다운로드 가능한 위치에. file:// 사용 시 모든 BE에 같은 경로로 배포.
  • JVM: BE의 JAVA_HOME이 JDK여야 (JRE 아님).
  • 보안: UDF는 BE 프로세스 권한으로 실행 — 신뢰된 코드만.
클러스터 적용 노트
  • UDF는 가능한 한 SQL/표준 함수로 풀고, 정말 필요한 경우만 도입. 운영 부담(JAR 배포·버전 관리)이 작지 않음.
  • 해시·암호화·검증 같은 컴팩트한 변환은 Python UDF가 빠르게 작성 가능하지만, 운영용 큰 트래픽은 Java UDF가 더 안정적.

SQL-5. 집계 · 윈도 함수

집계 함수

함수의미
COUNT(*) / COUNT(col)행 수 / NOT NULL 개수
COUNT(DISTINCT col)고유 값 (정확). 매우 큰 카디널리티는 비쌈
SUM / AVG / MIN / MAX표준
STDDEV / VARIANCE표준편차/분산
ANY_VALUE(col)그룹의 임의 한 값
ARRAY_AGG(col)배열로 모음
GROUP_CONCAT(col, sep)문자열로 이어붙임

근사 집계 (대규모)

-- 정확 distinct (느림)
SELECT count(distinct user_id) FROM events;

-- 근사 distinct (HLL, 매우 빠름. 오차 ~1%)
SELECT approx_count_distinct(user_id) FROM events;

-- 정확 distinct를 사전 집계하려면 BITMAP_UNION을 Aggregate 모델에서.
-- 자세히는 SQL-6. Bitmap·HLL 페이지 참조.

분위수 / 확률

SELECT percentile_approx(latency_ms, 0.5)  AS p50,
       percentile_approx(latency_ms, 0.95) AS p95,
       percentile_approx(latency_ms, 0.99) AS p99
FROM logs;

SELECT percentile_cont(0.5) WITHIN GROUP (ORDER BY x) FROM t; -- 정확

ROLLUP / CUBE / GROUPING SETS

SELECT dt, country, SUM(amt)
FROM orders
GROUP BY ROLLUP (dt, country);

GROUP BY GROUPING SETS ((dt, country), (dt), ());

-- 실제 어느 row인지 표시
SELECT GROUPING(dt) AS g_dt, GROUPING(country) AS g_co, ...

윈도 함수

-- 사용자별 최근 3건의 주문
SELECT *,
  ROW_NUMBER() OVER (PARTITION BY user_id ORDER BY updated_at DESC) AS rn
FROM orders
QUALIFY rn <= 3;

-- 누적 합 / 7일 이동평균
SELECT dt,
  SUM(amount) OVER (ORDER BY dt) AS cum_amt,
  AVG(amount) OVER (ORDER BY dt ROWS BETWEEN 6 PRECEDING AND CURRENT ROW) AS ma7
FROM daily;

-- 변화율
SELECT dt, amount,
  amount - LAG(amount, 1) OVER (ORDER BY dt) AS delta,
  amount / NULLIF(LAG(amount, 1) OVER (ORDER BY dt), 0) - 1 AS change_pct
FROM daily;

-- 분위수
SELECT *, NTILE(10) OVER (ORDER BY score) AS decile FROM users;

자주 쓰는 윈도 함수 정리

함수의미
ROW_NUMBER()순번 (동률도 다른 번호)
RANK() / DENSE_RANK()동률 처리 차이
LAG(col, n) / LEAD(col, n)이전/다음 n번째 값
FIRST_VALUE / LAST_VALUE윈도 첫/마지막 값
NTH_VALUE(col, n)n번째 값
NTILE(n)n분위 분할
CUME_DIST() / PERCENT_RANK()누적 확률 / 순위 비율

프레임 (ROWS / RANGE)

OVER (PARTITION BY x ORDER BY t
      ROWS BETWEEN 3 PRECEDING AND CURRENT ROW)

OVER (PARTITION BY x ORDER BY t
      RANGE BETWEEN INTERVAL 7 DAY PRECEDING AND CURRENT ROW)
클러스터 적용 노트
  • 대시보드의 COUNT(DISTINCT user)는 거의 항상 BITMAP/HLL로 대체. UV 정확도가 1% 이내면 충분한 경우 HLL이 압도적.
  • 윈도 함수 + PARTITION BY가 큰 경우 셔플이 무겁다. PARTITION BY 키와 테이블 분배 키를 일치시키면 BE 내 로컬 윈도로 동작.
  • GROUPING SETS는 큐브 작성에 강력. 단일 쿼리로 여러 차원 집계가 필요한 BI 시나리오에 적합.

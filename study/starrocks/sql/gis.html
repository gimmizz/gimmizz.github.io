SQL — GIS · 공간 함수

StarRocks는 OGC SFS 스타일의 기본 공간 함수를 지원한다. PostGIS 수준의 깊이는 아니지만 거리/포함/생성에는 충분.

생성

SELECT ST_Point(127.0, 37.5);                    -- 경도, 위도
SELECT ST_GeometryFromText('POINT(127.0 37.5)');
SELECT ST_GeometryFromText('LINESTRING(0 0, 1 1, 2 2)');
SELECT ST_GeometryFromText('POLYGON((0 0, 1 0, 1 1, 0 1, 0 0))');
SELECT ST_Circle(127.0, 37.5, 1000);             -- 반경(미터)

텍스트 변환

SELECT ST_AsText(geom);                          -- WKT
SELECT ST_AsBinary(geom);                        -- WKB
SELECT ST_GeometryFromText('POINT(127 37)');

측정·관계

SELECT ST_Distance_Sphere(lon1, lat1, lon2, lat2);  -- 미터 (lon/lat 직접)
SELECT ST_Contains(polygon, point);
SELECT ST_Within(point, polygon);
SELECT ST_Intersects(g1, g2);
SELECT ST_Overlaps(g1, g2);

예시 — 사용자 위치 ↔ 매장 매칭

CREATE TABLE store (
  store_id BIGINT,
  lon DOUBLE, lat DOUBLE,
  name VARCHAR(64)
)
PRIMARY KEY(store_id) DISTRIBUTED BY HASH(store_id);

CREATE TABLE visit (
  visit_id BIGINT,
  user_id  BIGINT,
  lon DOUBLE, lat DOUBLE,
  ts DATETIME
)
DUPLICATE KEY(visit_id) DISTRIBUTED BY RANDOM;

-- 1km 안에 있는 매장 찾기 (거리 기반 — 인덱스 최적화는 제한적)
SELECT v.visit_id, s.name,
       ST_Distance_Sphere(v.lon, v.lat, s.lon, s.lat) AS d_m
FROM visit v
JOIN store s
  ON ST_Distance_Sphere(v.lon, v.lat, s.lon, s.lat) < 1000
WHERE v.ts >= now() - INTERVAL 1 HOUR;

주의

  • SR의 GIS는 인덱스 가속이 약함. 큰 풀스캔 거리계산은 느릴 수 있다.
  • 지리좌표(lon/lat)와 평면좌표(x/y)를 혼동하지 말 것 — ST_Distance_Sphere는 위경도 기반.
  • 대용량 GIS 분석은 PostGIS / Sedona를 거쳐 SR로 결과만 가져오는 패턴이 일반적.
클러스터 적용 노트
  • "근접 매장 찾기" 같은 단발 쿼리는 SR로 충분. 도시별 매장 1만개 × 방문 100만건 정도면 OK.
  • 도시·구역 ID를 적재 시 미리 부여(geofence 결과)해 두면 GIS 함수 없이 단순 등호 join이 가능 — 이쪽이 보통 더 빠름.

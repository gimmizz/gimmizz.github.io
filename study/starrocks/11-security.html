5-4. 보안 — SSL · LDAP · RBAC

SSL/TLS — MySQL 프로토콜 암호화

# fe.conf
enable_ssl = true
ssl_keystore_location = /opt/sr/ssl/server.keystore
ssl_keystore_password = changeit

클라이언트 측에서 --ssl-mode=REQUIRED 등으로 강제 암호화. 내부망이라도 RBAC와 함께 적용 권장.

LDAP / Active Directory 통합

# fe.conf
authentication_ldap_simple_server_host = ldap.example.com
authentication_ldap_simple_server_port = 389
authentication_ldap_simple_bind_root_dn = "cn=admin,dc=example,dc=com"
authentication_ldap_simple_bind_root_pwd = "..."
authentication_ldap_simple_bind_base_dn  = "ou=people,dc=example,dc=com"
authentication_ldap_simple_user_search_attr = "uid"
-- LDAP 인증 사용자
CREATE USER 'jdoe' IDENTIFIED WITH authentication_ldap_simple AS 'cn=jdoe,ou=people,dc=example,dc=com';

RBAC — 역할 기반 권한

-- 역할
CREATE ROLE bi_reader;
CREATE ROLE etl_writer;
CREATE ROLE platform_admin;

-- 권한
GRANT SELECT ON ALL TABLES IN DATABASE mart TO ROLE bi_reader;
GRANT INSERT, SELECT ON DATABASE mart TO ROLE etl_writer;
GRANT USAGE ON CATALOG hive_prod TO ROLE bi_reader;
GRANT SELECT ON ALL TABLES IN ALL DATABASES IN CATALOG hive_prod TO ROLE bi_reader;
GRANT operate ON SYSTEM TO ROLE platform_admin;

-- 사용자 - 역할
GRANT bi_reader TO USER 'jdoe';
GRANT bi_reader, etl_writer TO USER 'svc_etl';

Row-level / Column-level 보안 (v3.x)

-- 컬럼 마스킹 정책
CREATE MASKING POLICY mask_email AS (val STRING)
RETURNS STRING ->
  CASE WHEN current_role() IN ('bi_reader')
       THEN regexp_replace(val, '(^.).+(@.+$)', '\\1***\\2')
       ELSE val END;

ALTER TABLE mart.users MODIFY COLUMN email SET MASKING POLICY mask_email;
-- 행 레벨 정책
CREATE ROW ACCESS POLICY country_filter AS (country STRING)
RETURNS BOOLEAN ->
  current_user() = 'svc_etl' OR country = 'KR';

ALTER TABLE mart.events ADD ROW ACCESS POLICY country_filter ON (country);

Audit Log

audit_loader 플러그인으로 Audit log를 StarRocks 자체 테이블로 적재. 누가 무엇을 조회했는지 SQL로 검증 가능.

클러스터 적용 노트
  • 처음부터 LDAP 통합 — 사용자별 비밀번호 관리 부담을 없애고, 퇴사 시 자동 비활성화.
  • 역할 5~7개 표준화: platform_admin / data_eng / etl_writer / bi_reader / analyst_advanced / read_only_partner 정도.
  • 외부 카탈로그 권한도 동일 역할에 묶을 것: 카탈로그 권한이 따로 놀면 사용자 혼란.
  • Audit log → 별도 마트로 일자별 적재: 컴플라이언스 대응에 필수.
